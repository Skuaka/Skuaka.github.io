简介

标准shell——bash

​ 在Linux中，总是作为/bin/sh安装的标准shell是GNU工具集中的bash，它是开源的并且可以被移植到几乎所有的类unix系统上。

​ 在大多数Linux发行版中，默认的shell程序/bin/sh实际上是对程序/bin/bash的一个连接。

​ 可以使用如下命令来查看bash的版本号

$/bin/bash --version

切换shell

​ 只需要直接执行需要的shell程序，例如/bin/bash，就能够运行新的shell并且改变命令提示符了。

创建脚本

​ #!字符告诉系统同一行上紧跟在它后面的那个参数是用来执行本文件的程序。

​ #字符后面的整行为注释

​ exit命令的作用是确保脚本程序能够返回一个有意义的退出码，以供其他脚本调用。0表示运行成功。

​ 查看一个文件是否是脚本的最好方法是使用file命令file skuaka

执行脚本

两种方法：

  1. 调用shell，把脚本文件名当做参数$ /bin/sh skuaka

  2. 把脚本当做可执行文件$ chmod +x skuaka

    如果skuaka在当前目录，则用./skuaka调用脚本（推荐习惯）

    如果想直接用skuaka调用脚本，修改环境变量(不推荐，有安全隐患)。

修改环境变量

​ 输入PATH=$PATH:.或者编辑.bash_profile文件，把前者添加到文件末尾，然后重新登入。

如果这个命令只供你本人使用，可以在自己的家目录中创建一个bin目录，并且将这个目录添加到自己的PATH变量中。

如果想让别人使用，可以将程序存放到/usr/local/bin或其他系统目录。

变量

  • 变量不需要声明，使用它们的时候自动创建。
  • 默认情况下，所有变量都被看做字符串并且以字符串来储存，即使被赋值为数字时也是如此。
  • 在变量名前面加一个$来访问它储存的内容
  • 为变量赋值时等号两边不能有空格。
  • read命令读取用户的输入为变量a赋值read a
  • 使用双引号将包含空白字符的参数引起来。
  • 双引号中的$变量在运行时会被替换为值，通过在$前面加上\字符以取消它的特殊含义。
  • 单引号中不会发生替换现象

函数

基本规则

  • 在一个脚本程序中执行另外一个脚本程序要比执行一个函数慢的多

  • 函数的定义方式

  • shell中不存在前向声明，要把所有函数定义都放在任何一个函数调用之前。

  • 当函数被调用时，脚本程序的位置参数($*,$@,$#,$1,$2等)会被替换为函数的参数，这就是读取传递给函数的参数的方法。当函数执行完毕后，这些参数会恢复为之前的值。

  • return让函数返回数字值

  • 让函数返回字符串值的常用方法是让函数将字符串保存在一个变量中，然后在函数结束之后使用这个变量

    还可以echo一个字符串并捕获其结果

  • 使用local关键字在shell函数中声明局部变量，如果一个局部变量名字和全局变量名字相同，在函数的作用范围内，前者就会覆盖后者。

shell常见命令

shell脚本内部可以执行两种命令：

  • 外部命令(external command): 可以在命令提示符中执行的普通命令
  • 内部命令(internal command): 内部命令是在shell内部实现的，不能作为外部程序被调用。（然而，大部分的内部命令同时也提供了独立运行的程序版本——属于POSIX规范）

常用命令：

  1. break 跳出循环

  2. 冒号(:)命令是一个空命令。

    它偶尔用于简化条件逻辑，相当于true的别名，while ：实现了无限循环。由于它是内置命令，所以运行的比true快，但是输出可读性差。

  3. continue 跳到下一次循环

  4. 点(.)命令，用于在当前shell中执行命令。

    ​ 通常，当一个脚本执行一条外部命令或脚本程序时，它会创建一个新的环境（一个子shell），命令将在这个行环境中执行，在命令执行完毕后，这个环境被丢弃，留下退出码返回给父shell。

    ​ 所以，当你要用一个脚本来为后续执行的命令设置环境时，这命令就非常有用，在这种意义上，点命令的作用类似于C/C++的#include指令。

    ​ 示例：假设有两个包含环境设置的文件，他们分别针对两个不同的开发环境。

    （1）为了设置老的环境，可以使用文件classic_set

    （2）对于新命令，使用文件lastest_set

    （3）可以通过将这些脚本程序和点命令结合来设置环境

  5. echo命令

    ​ 虽然，X/Open建议在现代shell中使用printf命令，但是使用echo方便。

    ​ 如何去掉换行符？

    • echo -n "str" -n参数去掉换行符

    • echo -e "str\c" -e参数确保启用了反斜线转义字符的解释(\c代表去掉换行符)

      老版本bash默认开启反斜线转义字符的解释，但最新版本的bash不一定开启。

    如果需要脚本兼容UNIX系统并且需要删除换行符，最好坚持使用printf命令。

    如果只运行在Linux和bash上，那么echo-n是不错的选择。

  6. eval命令，允许你对参数进行求值。示例如下

    它输出$foo，而

    输出10。因此，eval命令有点像一个额外的$，它给出一个变量的值的值。

  7. exec命令 有两种用法,第二种略。

    典型用法是将当前shell替换为一个不同的程序。

    exec wall "Thanks for all the fish"

    脚本中的这个命令会用wall命令替换当前的shell。脚本程序中exec命令后面的代码都不会执行，因为执行这个脚本的shell已经不存在了。

  8. exit n 命令

    • 使脚本以退出码n结束运行。

    • 在交互式shell命令提示符中输入exit以退出系统。

    • 如果脚本程序在退出时不指定退出状态，那么脚本中最后一条被执行的命令状态将被用作返回值。

    • 退出码

      0 表示成功

      1-125 脚本程序可以使用的错误代码

      126 文件不可执行

      127 命令未找到

      128及以上 出现一个信号

    • 精简的退出码示例：如果文件存在则返回0，否则1。

      [ -f .profile ] && exit 0 || exit 1

  9. export命令 导出命令

    作为export参数的变量，就可以在子shell中使用。而不会因为调用子shell而清除变量的值。

    set -a 或set -o allexport命令将导出它之后声明的所有变量。

  10. expr命令将它的参数当做一个表达式来求值，如

    x=$(expr $x + 1) 使x取值为表达式 expr $x + 1的执行结果。也可以用反引号(`)字符将表达式括起来。

    expr可以用于很多算数表达式。比大小、加减乘除、取模取余。比较特殊的两个

  11. printf命令与C/C++类似，不一样的是不需要括号、不支持浮点数，因为shell中所有算数运算都是按照整数来进行运算的

  12. return命令，使函数返回一个数值。

  13. set命令，为shell设置参数变量。

    这样就可以单独提取date里面的月份。

    还有其他用处，略。

  14. shift命令，将所有参数都左移一个位置，$2变成$1,$3变成$2，$* $@ $#也相应地改变。（$0是脚本名字，不变）。

    如果shift后面带有数值参数n，则表明左移n位。

  15. trap命令，指定在收到信号后将要采取的行动。trap command signal

  16. unset命令，从环境中删除变量或函数（不能删除shell本身的只读变量）
