CCIE NAT 地址转换协议

NAT 地址转换协议

优缺点

优点
在地址重叠时提供解决方案

在网络发生变化时避免重新编织

最大作用为可将私网地址转换为公网地址，提升了ip地址空间的使用率，这也是ipv4至今还未耗尽的原因

缺点

增加交换延迟

无法进行端到端ip跟踪

有些应用程序无法正常运行

网络地址转换类型

静态NAT：在本地地址和全局地址直接进行一对一的映射。必须为网络中的每台主机提供一个公有ip地址。

动态NAT：将未注册的ip地址映射到注册ip地址池中的一个地址。必须有足够多的公有ip地址，让连接到因特网的主机都能够同时发送和接收分组。

NAT重载：最常用的NAT类型

也被称为端口地址转换(PAT)

利用源端口将多个非注册ip地址映射到一个注册ip地址(多对一)

使用传输层端口号来标识主机，因此理论上最多可让大约65 000台主机共用一个公有ip地址

本地地址：进行NAT转换前的地址

内部本地地址：转换前的内部源地址，主机的私网地址

外部本地地址：转换后的目标地址，与ISP相连的路由器接口的地址，通常为共有地址

全局地址：进行NAT转换后的地址，通常为因特网共有地址

内部全局地址：转换后的源地址

外部全局地址：转换前的外部目标地址

配置方式

接口角色：
inside：连接在内部广播域
outside：连接公网

转换接口角色
int xx
ip nat inside/outside

静态NAT

ip nat inside source static 内部ip 外部ip

动态NAT

ip nat pool todd 170.168.2.3 170.168.2.254 netmask 255.255.255.0

创建一个地址池todd，用于将全局地址分配给主机

ip nat inside source list 1 pool todd

将与access-list 1匹配的ip地址转换为ip地址池todd中的一个可用地址

PAT(NAT重载)

与动态NAT相比，PAT地址池只包含一个ip地址，且配置命令末尾包含关键字overload

ip nat pool globalnet 170.168.2.1170.168.2.1 netmask 255.255.255.0

ip nat inside source list 1 pool overload

(重复使用，如果地址池有其他地址，也可以使用它，适合大型网络，必须重载多个公有ip地址)

验证与修改

show ip nat translation

debug ip nat

show ip nat statistics

命令claer ip nat translations 清除转换条目，可以使用全局地址、本地地址、tcp(udp)端口、*（星号，清除所有转换条目）

只要NAT表中有包含地址池中地址的转换条目，思科ios就不允许修改或删除相应的地址池

转换条目超时时间：默认为24小时，可以使用命令ip nat translation timeout修改。如果定时器到期，相应的转换条目将从NAT表中删除，动态分配给它的地址将归还给地址池。

CCIE MPLS VPN

VPN

虚拟的私有网络，通过组建虚拟的专网实现数据的安全传递。

VPN的分类

OSI 参考模型分类

  • 应用层VPN：开源的第三方VPN软件（open vpn）
  • 传输层VPN：SSL VPN（安全套字节层 VPN）主要用于HTTP的安全加固和防护。
  • 网络层VPN：IPSEC VPN（ip安全VPN）、DM-VPN（动态多点VPN，思科私有）
  • 数据链路层VPN：L2TP（层2隧道VPN）、PPTP（点到点隧道VPN）微软思科联合开发
  • 物理层VPN：专线

实际应用分类

  1. site to site：站点到站点：L2L（lan to lan）

    IPSEC VPN、DM-VPN、MPLS VPN

  2. remote vpn：远程接入VPN

    L2TP、PPTP、SSL VPN、EZ-VPN（思科私有）、anyconnect VPN（任连VPN）

MPLS VPN

MPLS VPN是在BGP peer之间建立（peer to peer VPN），利用了BGP的对等体共享了企业内网路由信息，构建专有通路，实现安全的通讯。

MPLS VPN术语

  • P-NETWORK：运营商核心网络

  • PE 设备：运营商边界设备

  • P 设备：运营商核心设备

  • C-NETWORK：客户网络

  • CE 设备：客户网络边界路由器

  • VRF：虚拟路由转发表

    路由器拥有一个全局路由表和若干个VRF，每个VRF对应到一个C-NETWORK。

    作用：存储不同 C-NETWORK 的路由，保障这些路由不影响全局路由表，而且不会冲突，且相互独立。

    • VRF 的 RD 值：路由区分值

      RD 值书写格式：XX:YY

      不同 VRF 的路由进入BGP 后，利用 RD值进行区别。

      MPLS VPN peer 之间建立的是 MP-BGP peer 关系，MP-BGP 提供专门的转发表，存储来自不同 VRF的路由。

    • VRF 的 RT 值：路由目标值

      RT 值书写格式：XX:YY

      RT 值分为导入和导出两个方向：

      导入值是用来确认是否学习远端的路由

      导出值是路由被共享时携带

      导入和导出值可以是多个

  • MP-BGP VPNv4 更新：即 MP-BGP 的路由更新

MPLS VPN 的双标签

栈底标签：PE 设备上 MP-BGP 给予的，用来帮助数据寻找正确的 VRF路由

栈顶标签：MPLS LDP 给予的，用来穿越P-NETWORK。

安全加密

GET-VPN（隧道加密传输VPN，思科私有）专门在MPLS VPN 基础上加密数据。

MPLS VPN 的配置流程

步骤1：完成 P-NETWORK 的配置

  1. 完成 P-NETWORK 底层协议的启用（OSPF 或 ISIS）

    设备loopback口必须可达，因为BGP MPLS MP-BGP 需要它。

  2. 完成 P-NETWORK 中 P 与 PE 设备的 MPLS 的配置

    确保设备间的 LDP 邻居正常建立。

bgp的防环机制

iBGP-水平分隔原则

BGP对等体路由器收到iBGP路由，不会发送给其他iBGP对等体路由器。

但是纯水平拓扑上该原则会导致路由不被传递学习。两个解决方案

全互联拓扑解决方案：所有路由器间都建立iBGP关系。（垃圾方案）

eBGP

基于AS PATH信息的检测。接收到来自于eBGP peer的路由更新，更新所带AS-PATH 信息包含了当前设备AS号，则该路由更新忽略。

特殊拓扑下路由反射器的防环机制

originate id：起源id。路由器将设备的RID设置为起源ID。起源ID和当前设备RID一样，则路由更新不学习。

cluster id：簇id。管理员在路由反射器上设置，路由更新携带的簇ID一致，则路由更新不学习。

路由更新来自于iBGP peer时才考虑同步和下一跳。

对于当前启用BGP的设备，路由更新对应的网络底层协议也必须能学习到，这才叫同步。同步是为了避免路由黑洞。

来自于iBGP peer的路由更新，如果下一跳不可达，路由更新不学习。

路由源于AS外，通告者是eBGP peer，下一跳就是eBGP peer地址。

路由源于iBGP peer，通告者是iBGP peer，下一跳就是iBGP peer地址。

端口状态

Blocking（阻塞状态）：此时，二层端口为非指定端口，也不会参与数据帧的转发。该端口通过接收BPDU来判断根交换机的位置和根ID，以及在STP拓扑收敛结束之后，各交换机端口应该处于什么状态，在默认情况下，端口会在这种状态下停留20秒钟时间。

Listening（侦听状态）：生成树此时已经根据交换机所接收到的BPDU而判断出了这个端口应该参与数据帧的转发。于是交换机端口就将不再满足于接收BPDU，而同时也开始发送自己的BPDU，并以此通告邻接的交换机该端口会在活动拓扑中参与转发数据帧的工作。在默认情况下，该端口会在这种状态下停留15秒钟的时间。

Learning(学习状态)：这个二层端口准备参与数据帧的转发，并开始填写MAC表。在默认情况下，端口会在这种状态下停留15秒钟时间。

Forwarding（转发状态）：这个二层端口已经成为了活动拓扑的一个组成部分，它会转发数据帧，并同时收发BPDU。

Disabled（禁用状态）：这个二层端口不会参与生成树，也不会转发。

JSON

JavaScript 对象表示法

JSON 是存储和交换文本信息的语法。类似 XML。

JSON 比 XML 更小、更快，更易解析。

JSON 使用 JavaScript 语法来描述数据对象，但是 JSON 仍然独立于语言和平台。JSON 解析器和 JSON 库支持许多不同的编程语言。

JSON - 转换为 JavaScript 对象

JSON 文本格式在语法上与创建 JavaScript 对象的代码相同。

由于这种相似性，无需解析器，JavaScript 程序能够使用内建的 eval() 函数，用 JSON 数据来生成原生的 JavaScript 对象。

为什么使用 JSON？

对于 AJAX 应用程序来说，JSON 比 XML 更快更易使用：

  1. 使用 XML
  • 读取 XML 文档
  • 使用 XML DOM 来循环遍历文档
  • 读取值并存储在变量中
  1. 使用 JSON
  • 读取 JSON 字符串
  • 用 eval() 处理 JSON 字符串

JSON 语法规则

JSON 语法是 JavaScript 对象表示法语法的子集。

  • 数据在名称/值对中
  • 数据由逗号分隔
  • 花括号保存对象
  • 方括号保存数组

JSON 值可以是：

  • 数字（整数或浮点数）
  • 字符串（在双引号中）
  • 逻辑值（true 或 false）
  • 数组（在方括号中）
  • 对象（在花括号中）
  • null

JSON 解析器

提示：eval() 函数可编译并执行任何 JavaScript 代码。这隐藏了一个潜在的安全问题。

使用 JSON 解析器将 JSON 转换为 JavaScript 对象是更安全的做法。JSON 解析器只能识别 JSON 文本，而不会编译脚本。

在浏览器中，这提供了原生的 JSON 支持，而且 JSON 解析器的速度更快。

较新的浏览器和最新的 ECMAScript (JavaScript) 标准中均包含了原生的对 JSON 的支持。
