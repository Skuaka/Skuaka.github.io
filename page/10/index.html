CCIE 跨域的多播

跨域的多播

源和组成员在不同的AS，需要传递多播数据流。

跨域多播的问题和挑战

组成员无法发现源、源无法发现组成员、多播树无法建立。

MSDP 多播源发现协议

MSDP 是配置在不同 AS RP 上的，它会建立基于 TCP 的对等体关系，利用自身的 SA（源关联）消息共享”源地址信息、多播组地址信息、RP 信息”给对端 RP。

MSDP 配置流程

步骤1：RP上配置新的 loopback 接口，并且宣告进 BGP。

步骤2：RP上完成 MSDP 的配置

排错手段

ip 路由疑难解析

多播排错

ping 多播组地址 不同

  1. 确认发现组成员

    show ip igmp group

    查看组地址、本地接口、报告者 是否出现

    错点

    • 没有模拟成组成员，缺乏接口下：ip igmp join-group <a.b.c.d>
    • 路由器接口没有加入PIM，缺乏接口下:ip pim <x>
    • 连通性问题

  2. 确认稀疏模式获取 RP 信息

    show ip pim rp mapping 查看 RP 映射，查看 RP 的结果

    show ip pim rp 查看多播组和 RP 的关联

    错点

    • 没有通过静态 RP 指明 RP 地址
    • BSR 的 C-BSR 或 C-RP 配置问题
    • auto-rp 的配置问题
    • 缺乏auto-rp 侦听的配置
    • 设备间缺乏必要的PIM 邻居：show ip pim neighbor
      • 设备缺乏多播路由功能和接口忘记加入PIM
      • 接口忘记激活、错误的安全策略挡住了PIM消息
    • 设备无法正常抵达 RP 地址，所以不显示获取的 RP 消息
      • 路由层面问题、忘记宣告、忘记重分发、错误的过滤

  3. RPF 检测未通过

    • 缺乏必要的更新源路由
    • 忘记宣告、忘记重分发、错误的路由更新过滤

  4. 是否是跨域多播、缺乏必要的配置和连通性

    • 缺乏必要的 MSDP 配置和 MSDP 对等体关系

      show run | sec ip msdp

      show ip msdp summary

      MSDP peer 地址不可达

      MSDP 链接源和起源 ID 设置错误

    • RPF 检测不通过

  5. 错误的安全策略

防火墙

防火墙的分类

软件防火墙：省查木马或病毒：保护的是系统

硬件防火墙

华为、天融信

硬件防火墙的部署

防火墙的基础知识

防火墙工作模式

路由模式：（默认模式）防火墙读取 IP 包头，进行网络层的保护，支持动态路由协议和静态路由。

交换模式：

防火墙的多模式和单模式

多模式支持虚拟防火墙，即把一台防火墙配置成多台逻辑防火墙。

单模式是默认模式。

防火墙接口的类型

inside：内部接口

通常连接的是信任网络即内网

outside：外部接口

通常连接的是非信任公共网络，比如 ISP 或 其他组织网络

DMZ：非军事区域

通常数据中心放在 DMZ 区域

防火墙接口等级和访问规则

inside 接口安全等级：100

outside 接口安全等级：0

DMZ 接口安全等级：50、0

高等级接口接收流量穿越防火墙从低等级接口发送出去，必须经过 NAT。

低等级接口接收流量穿越防火墙从高等级接口发送出去，默认不允许。

除非是返回流量或被管理员手动放行。

相同安全级别的接口相互通讯，默认不允许通讯，除非管理员放行。

CCIE MPLS VPN

VPN

虚拟的私有网络，通过组建虚拟的专网实现数据的安全传递。

VPN的分类

OSI 参考模型分类

  • 应用层VPN：开源的第三方VPN软件（open vpn）
  • 传输层VPN：SSL VPN（安全套字节层 VPN）主要用于HTTP的安全加固和防护。
  • 网络层VPN：IPSEC VPN（ip安全VPN）、DM-VPN（动态多点VPN，思科私有）
  • 数据链路层VPN：L2TP（层2隧道VPN）、PPTP（点到点隧道VPN）微软思科联合开发
  • 物理层VPN：专线

实际应用分类

  1. site to site：站点到站点：L2L（lan to lan）

    IPSEC VPN、DM-VPN、MPLS VPN

  2. remote vpn：远程接入VPN

    L2TP、PPTP、SSL VPN、EZ-VPN（思科私有）、anyconnect VPN（任连VPN）

MPLS VPN

MPLS VPN是在BGP peer之间建立（peer to peer VPN），利用了BGP的对等体共享了企业内网路由信息，构建专有通路，实现安全的通讯。

MPLS VPN术语

  • P-NETWORK：运营商核心网络

  • PE 设备：运营商边界设备

  • P 设备：运营商核心设备

  • C-NETWORK：客户网络

  • CE 设备：客户网络边界路由器

  • VRF：虚拟路由转发表

    路由器拥有一个全局路由表和若干个VRF，每个VRF对应到一个C-NETWORK。

    作用：存储不同 C-NETWORK 的路由，保障这些路由不影响全局路由表，而且不会冲突，且相互独立。

    • VRF 的 RD 值：路由区分值

      RD 值书写格式：XX:YY

      不同 VRF 的路由进入BGP 后，利用 RD值进行区别。

      MPLS VPN peer 之间建立的是 MP-BGP peer 关系，MP-BGP 提供专门的转发表，存储来自不同 VRF的路由。

    • VRF 的 RT 值：路由目标值

      RT 值书写格式：XX:YY

      RT 值分为导入和导出两个方向：

      导入值是用来确认是否学习远端的路由

      导出值是路由被共享时携带

      导入和导出值可以是多个

  • MP-BGP VPNv4 更新：即 MP-BGP 的路由更新

MPLS VPN 的双标签

栈底标签：PE 设备上 MP-BGP 给予的，用来帮助数据寻找正确的 VRF路由

栈顶标签：MPLS LDP 给予的，用来穿越P-NETWORK。

安全加密

GET-VPN（隧道加密传输VPN，思科私有）专门在MPLS VPN 基础上加密数据。

MPLS VPN 的配置流程

步骤1：完成 P-NETWORK 的配置

  1. 完成 P-NETWORK 底层协议的启用（OSPF 或 ISIS）

    设备loopback口必须可达，因为BGP MPLS MP-BGP 需要它。

  2. 完成 P-NETWORK 中 P 与 PE 设备的 MPLS 的配置

    确保设备间的 LDP 邻居正常建立。

  3. 完成 PE 设备的 BGP 与 MP-BGP 的启用配置

    注意！！MPLS一定要配置在 MP-BGP 之前，否则 MPLS VPN 无法正常工作。

  4. 完成 P-NETWORK 中 PE设备上 VRF 的创建于配置

UNTAGE 对 MPLS VPN 通讯的影响

栈顶标签对应的配置出站标签如果是UNTAGE，那么会导致所有标签被移除，数据变回IP包，很可能会面对路由黑洞或者是无法路由的局面，数据包丢包。

PE 上 OSPF 的问题

新的 OSPF 层次化架构

MPLS VPN 中 OSPF 将整个自治系统分成了：

超级骨干层（P-NETWORK）、骨干层、常规层。

PE 设备 OSPF 的限制

PE 设备和 CE 设备间 OSPF 的区域最好是区域0，否则会出现路由不学习问题，需要虚链路解决该问题。

OSPF 通过 MPLS VPN 共享路由后的类型
  1. 相同区域路由通过 MPLS VPN 共享，最后结果是O IA
  2. 不同区域路由通过 MPLS VPN 共享，最后结果是O IA
  3. 外部路由通过MPLS VPN 共享，最后结果依旧是O E2
利用 sham-link（伪链路）还原相同区域的路由

MPLS VPN 排错

C-NETWORK 之间无法正常通讯

  1. 当前设备是否学习对端 C-NETWORK 路由

    • CE 是否将路由共享给当前设备，导致当前设备缺乏必要路由。

      动态路由协议邻接关系没建立，错误的路由更新过滤策略。

    • CE 与 PE 间是否正常交换路由更新

      错误的邻接关系、PE 和 CE 间为满足 MPLS VPN 对 OSPF 的要求

      PE 上忘记把 BGP 重分发进底层 VRF

    • PE 设备没有从对端 PE 学习路由

      show ip bgp vpnv4 all 查看MPBGP 转发表，找对端的路由条目。

      MP-BGP 的对等体关系有问题

      show ip bgp vpnv4 all summary 查看对等体关系：

      忘记指定对等体、忘记设置更新源、忘记激活 MP-BGP peer 关系，peer 地址不可达，错误的安全策略导致 BGP TCP 连接无法建立。

      show ip vrf 当前设备缺乏 vrf 的配置

      AS内缺乏路由反射器的配置

    • 对端 PE 设备无法从 CE 设备获取路由信息

  2. 标签问题

    PE 上查看 BGP 路由的下一跳，确认栈顶标签关联的目标。

    show ip bgp vpnv4 all 通过该命令查看下一跳

    错点

    • 标签路径不关联

      出现了不必要的 UNTAGE，缺乏 loopback 下 ip OSPF network point-to-point

      出现了不必要的 NO LABEL，没有远端 LDP 邻居共享标签（当前设备是否缺乏必要的 LDP 邻居，对方没有学习目标网络路由）

  3. ping 包源对方是否可达

  4. 错误的访问控制策略

跨域MPLS VPN 解决方案详解

方法一：option A：背靠背：back to back

  1. 步骤1：ASBR 上创建 VRF，设定 RD 和 RT，将直连接口划入 VRF。

    R3、R4 创建 VRF 的原因：

  • 利用 VRF 接收来自路由反射器的路由更新，当前 ASBR 设备学习本 AS 站点路由。

  • 创建基于 VRF 的 EBGP PEER 关系，帮助当前 ASBR 设备学习对端 C-NETWORK 路由，并且装载进 VRF 表。

    注意！！ PE 上启用 VRF，切记要将 VRF 和 BGP 相互做重分发，如果 PE 和 CE 间是 BGP，那么来自 CE 的路由更新直接进 PE 的 MP-BGP 转发表。

  • ASBR 创建 VRF 给对端 C-NETWORK 路由新的栈底标签。

方法一如何传递数据

方法一的特点：ASBR 间只有传统的 EBGP peer，ASBR 间没有 MPLS，ASBR 间数据最后是 IP 形式穿越。

方法二：option B 2a

步骤一：ASBR上关闭 RT 过滤功能，在不创建 VRF 的情况下学习 MP-BGP 路由更新，达到学习各自 AS 内部路由的目的

此时栈底标签也跟着共享过来了

步骤二：ASBR 间利用直连网络建立 MP-BGP 关系

方法三：option B 2b

步骤1：ASBR 自身关闭 RT 筛选

步骤2：ASBR 利用静态路由指明抵达对端 ASBR 的 loopback 0 口地址

步骤3：ASBR 间启用 MPLS

如上步骤是考虑到 ASBR 间可能非直连，存在路由黑洞，所以利用 MPLS 的标签，关联帮助数据穿越黑洞。

步骤4：ASBR 间利用环回口建立 MP-BGP EBGP peer

步骤5：将 ASBR 上的静态路由重分发进 BGP

方法四：option C

步骤1：ASBR 间建立 BGPv4 EBGP peer，并且给对方共享标签信息，对对方设置权重值为1.

步骤2：ASBR 上将 PE 和 RR 的 lo0 宣告进 BGP

如上步骤是为了 ASBR 交换各自 AS 内路由反射器及 PE 的路由信息，

ASBR 一定要对 AS 内设备配置 NEXT-HOP-SELF。

步骤3：两个不同的 AS 利用各自路由反射器 lo0 建立 MP-BGP EBGP peer 关系，针对对端做 next-hop-unchange。

步骤4：ASBR 上把来自于对端 AS 的路由重分发进底层 ISIS，将原本被 BGP 学习的路由带入 ISIS，使得当前 AS 的 MPLS 可以分配栈顶标签。

方法四特点：不同 AS 路由反射器直接建立 MP-BGP 对等体关系，使得不同 AS 的 PE 似乎和对端是直接交换路由更新。

如何共享不同站点的路由

  1. R2 与 R5 建立 MP-BGP peer，共享 VPNv4 路由更新

    R3 R4 利用直连建立了 eBGP peer、R3 R4将 R1 R2 R5 R6 ——————没记

  2. 数据传递时标签的问题：顶部标签、底部标签

    LDP 对本地直连、IGPS 协议所学习的路由分配标签

    LDP 无法对 BGP 路由分配标签

CCIE NAT 地址转换协议

NAT 地址转换协议

优缺点

优点
在地址重叠时提供解决方案

在网络发生变化时避免重新编织

最大作用为可将私网地址转换为公网地址，提升了ip地址空间的使用率，这也是ipv4至今还未耗尽的原因

缺点

增加交换延迟

无法进行端到端ip跟踪

有些应用程序无法正常运行

网络地址转换类型

静态NAT：在本地地址和全局地址直接进行一对一的映射。必须为网络中的每台主机提供一个公有ip地址。

动态NAT：将未注册的ip地址映射到注册ip地址池中的一个地址。必须有足够多的公有ip地址，让连接到因特网的主机都能够同时发送和接收分组。

NAT重载：最常用的NAT类型

也被称为端口地址转换(PAT)

利用源端口将多个非注册ip地址映射到一个注册ip地址(多对一)

使用传输层端口号来标识主机，因此理论上最多可让大约65 000台主机共用一个公有ip地址

本地地址：进行NAT转换前的地址

内部本地地址：转换前的内部源地址，主机的私网地址

外部本地地址：转换后的目标地址，与ISP相连的路由器接口的地址，通常为共有地址

全局地址：进行NAT转换后的地址，通常为因特网共有地址

内部全局地址：转换后的源地址

外部全局地址：转换前的外部目标地址

配置方式

接口角色：
inside：连接在内部广播域
outside：连接公网

转换接口角色
int xx
ip nat inside/outside

静态NAT

ip nat inside source static 内部ip 外部ip

动态NAT

ip nat pool todd 170.168.2.3 170.168.2.254 netmask 255.255.255.0

创建一个地址池todd，用于将全局地址分配给主机

ip nat inside source list 1 pool todd

将与access-list 1匹配的ip地址转换为ip地址池todd中的一个可用地址

PAT(NAT重载)

与动态NAT相比，PAT地址池只包含一个ip地址，且配置命令末尾包含关键字overload

端口状态

Blocking（阻塞状态）：此时，二层端口为非指定端口，也不会参与数据帧的转发。该端口通过接收BPDU来判断根交换机的位置和根ID，以及在STP拓扑收敛结束之后，各交换机端口应该处于什么状态，在默认情况下，端口会在这种状态下停留20秒钟时间。

Listening（侦听状态）：生成树此时已经根据交换机所接收到的BPDU而判断出了这个端口应该参与数据帧的转发。于是交换机端口就将不再满足于接收BPDU，而同时也开始发送自己的BPDU，并以此通告邻接的交换机该端口会在活动拓扑中参与转发数据帧的工作。在默认情况下，该端口会在这种状态下停留15秒钟的时间。

Learning(学习状态)：这个二层端口准备参与数据帧的转发，并开始填写MAC表。在默认情况下，端口会在这种状态下停留15秒钟时间。

Forwarding（转发状态）：这个二层端口已经成为了活动拓扑的一个组成部分，它会转发数据帧，并同时收发BPDU。

Disabled（禁用状态）：这个二层端口不会参与生成树，也不会转发。

CCIE IPSEC VPN

IPSEC VPN

Ip security vpn，是一个工作在网络层的 VPN 技术。

数据在公共网络上面临的风险

  1. 不加密，明文传递
  2. 没有完整性保护，数据容易被篡改
  3. 没有对等体认证，容易受到中间人攻击
  4. 抵赖

IPSEC VPN 解决的问题

  1. 对等体认证，数据发送之前必须经过认证且通过

    口令认证、数字证书（计算机世界的身份证）、加密随机数

  2. 数据加密

    des、3des、aes

  3. 数据完整性保护

    HMAC：哈希消息认证码

  4. 数字签名，防抵赖

    DSA（RSA 公司开发）

KEY：密匙

KEY 是一个工具，可以参与算法作为变量出现，也可以参与认证作为口令出现。

对称密钥、非对称密钥

（加密算法中的 KEY 的理解）

  1. 对等密钥及对称密钥加密算法

    加密解密密钥是同一把

    坏处：不够安全、Key 的共享存在风险

    好处：简单高效

  2. 非对称密钥及非对称密钥算法

    非对称密钥是一对KEY，公钥和密钥。

    公钥最后会公开，密钥不公开。

    公钥加密的数据，只有私钥可以解开。

    私钥加密的数据，只有公钥可以解开。

    坏处：复杂，数据会变庞大

    好处：安全可靠

IPSEC VPN 的加密密钥

非对称密钥共享是用对称密钥及算法保护

数据本身是用非对称密钥及算法保护

密钥产生过程和共享方法

DH 算法负责产生密钥和共享密钥

IPSEC VPN 的关键组件

ISAKMP：互联网安全关联密钥管理协议

  • ISA：通讯双方建立了安全连接就叫做一个 SA 成功了。

    IPSEC VPN 的 SA 有两个：一个是保护 VPN 的，一个是保护数据的。

  • KMP：KMP 是通过 IKE（互联网密钥交换机制）完成的，IKE 专门负责产生密钥，共享密钥。

    IKE 分两个阶段：阶段1产生和共享VPN自身所需的密钥，阶段2利用阶段1的密钥算出新密钥，用于保护用户数据。

ESP 和 AH

ESP 和 AH 都是 VPN 头部信息的缩写

帧头、包头、段头、数据、帧尾

数据经历 VPN 保护后变化成如下：

帧头、VPN 头、包头、段头、数据、帧尾

VPN 头就分两种：

  1. ESP：封装安全静荷

    该头负责保护数据的完整性，同时加密数据

  2. AH：认证头

    该头只负责保护数据完整性，不加密数据

IPSEC VPN 的模式

  1. 隧道模式（tunnel）

    VPN 网关通过一个隧道模式的 VPN 连接，保护身后所有的通讯数据。

  2. 传输模式（transport）

    通讯双方自身建立 VPN 连接，保护通讯数据。

区别：

隧道模式的新包头，目的 IP 和源 IP 是 VPN 的对等体地址。

传输模式的新包头，目的 IP 和源 IP 复制了原有的 IP 地址。

MPLS详解

multiple protocol label switch（多协议标签交换）

MPLS作用

MPLS是一种数据交换机制，基于标签库和数据携带的标签。

进行数据转发，同时支持多种协议。

MPLS在实际工作中，能实现如下两个基本功能：

  1. 数据包的高速转发
  2. 减少参与BGP的设备数量、减少对等体关系、减少不必要的路由学习

MPLS术语

label 标签

总长 32bit 由如下几个部分组成：

前20bit 是标签取值范围

第21bit 是栈底位，栈底位为1则表明该标签已经是最后一个

第22~24bit 是体验位，可以将标签对应的数据分成8个级别，给予不同的传输策略。

最后8bit 是TTL，限制携带标签的数据传递范围。

MPLS 模式

帧模式：frame mode（主流模式），标签被插在帧头的后面，包头前面。

信令模式：cell mode 标签被插在信令头后面。

LSR：标签交换路由器

启用了MPLS的路由器统称LSR。

入站 LSR （inbound LSR）：接收到 IP 数据包，压入标签，传递给 MPLS 网络。

链路中 LSR （link LSR）：接收到携带标签的数据，基于标签进行数据转发。

出站 LSR （outbound LSR）：负责将标签移除，并且变回 IP 数据报文。

压入、交换、移除、弹出

压入：入站LSR将标签压入数据。

交换：链路中LSR基于标签库，进行标签的置换、数据的传递。

移除：出站LSR将数据携带的所有标签全部删除。

弹出：出站LSR将数据携带的栈顶标签弹出，其他标签保留。

LSP：链路标签路径

是MPLS设备传递数据所需的路径。（即路由）

LSP是通过标签的关联产生的。

标签分配、分发、关联、维护协议

TDP：标记分发协议、思科私有

LDP：标签分发协议、公有公开

LDP详解

LDP作用

  1. 利用 hello 消息建立 LDP 邻接关系。
  2. 基于CEF给本地网络分配本地标签。
  3. 将网络对应的标签分发共享给邻居
  4. 基于底层协议完成本地标签与远程共享标签的关联，最终形成LSP。
  5. 负责监控LSP的有效性，并且维护它。

LDP邻接关系的建立

LDP 邻居的发现，通过hello消息，基于UDP，目标端口646。

LDP 利用TCP，目标端口646完成邻接关系建立。

LDP可以设定RID，RID扮演更新源，更新源会成为对端TCP的目标接收地址，所以更新源必须可达。

LDP标签的分配

除BGP路由，LDP可以给本地直连网络以及非直连网络分配标签。

LDP标签的共享

两种形式：下游主动、上游主动。

实际上标签的分享是给所有的邻居，不遵循所谓的上下游。

LDP标签的关联

LDP永远将路由路径下一跳共享的标签与对应网络的本地标签关联。

boundaddress：绑定地址，参与MPLS设备的IP会成为绑定地址和，会跟着本地标签一起共享给LDP邻居，邻居根据底层协议下一跳地址和绑定地址的判断来确认哪个LDP邻居是下一跳，再绑定。

LDP的特殊标签

  • UNTAGE：未标记

出现原因：底层是ospf,本地网络无法找到对应的远程标签，LDP给网络分配untage。

处理方式：目标网络远程标签如果是untage，那么LSR会将所有标签移除，向外传递。

  • POP：倒数第二跳标签

出现原因：当前路由器给本地直连网络分配的本地标签就是POP。

处理方式：如果出站标签是POP，那么LSR 会将栈顶标签弹出。

意义：提高数据转发效率，减少一次标签库查找。

LDP的两张表

LIB：标签信息库

目标网络本地标签和学习到的远程标签存储在这里。（未关联）

LFIB：标签转发信息库

标签关联后的结果 即 LSP 存储在这里。
