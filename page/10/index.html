1直连进路由选择协议

redistribute connect metric 1

router rip

router eigrp 100

router os 1

re ei 100 subnets metric-type 1 (+metric 值）

CCIE 防火墙

防火墙

防火墙的分类

硬件防火墙：流量的省查及访问的控制：保护的是网络

软件防火墙：省查木马或病毒：保护的是系统

应用防火墙：WAF（WEB 应用防火墙）、APP 防火墙

硬件防火墙

checkpoint（诺基亚）

netscreen（詹博、Juniper）

PIX、ASA（Cisco）SDN（安全自防御网络、软件定义的网络）

华为、天融信

硬件防火墙的部署

网络的边缘部署防火墙，一般放在 PPPOE 拨号设备的身后，关键节点之间。

防火墙的基础知识

防火墙工作模式

路由模式：（默认模式）防火墙读取 IP 包头，进行网络层的保护，支持动态路由协议和静态路由。

交换模式：

防火墙的多模式和单模式

多模式支持虚拟防火墙，即把一台防火墙配置成多台逻辑防火墙。

单模式是默认模式。

防火墙接口的类型

inside：内部接口

通常连接的是信任网络即内网

outside：外部接口

通常连接的是非信任公共网络，比如 ISP 或 其他组织网络

DMZ：非军事区域

通常数据中心放在 DMZ 区域

防火墙接口等级和访问规则

inside 接口安全等级：100

outside 接口安全等级：0

DMZ 接口安全等级：50、0

高等级接口接收流量穿越防火墙从低等级接口发送出去，必须经过 NAT。

低等级接口接收流量穿越防火墙从高等级接口发送出去，默认不允许。

除非是返回流量或被管理员手动放行。

相同安全级别的接口相互通讯，默认不允许通讯，除非管理员放行。

  3. 注入静态路由

  4. 防火墙上配置 ACL

  5. 防火墙上配置 NAT

    1. 创建对象网络，添加允许转化的内网信息

      防火墙可以创建多个对象网络，一个对象网络对应到一个部门。

    2. 创建对象组，将对象网络关联并且设置 NAT 转化语句。

      ！！补充知识

      NAT：网络地址转换

      静态 NAT：管理员指定的一对一转化

      动态 NAT：路由器自动的完成一对一转化

      端口 NAT：PAT 多对一的转化，利用端口号区分开

      ！！注意

      防火墙默认不允许 PING 包流量从低等级接口穿越防火墙抵达高等级接口。

    3. 内网服务发布到外网

      1. 定义对象网络，把服务器地址放入

      2. 完成静态 NAT 的配置发布服务

      3. 利用 ACL 放行来自外网主动访问内网服务的流量

route-map、prefix-list、filter-list（筛选列表）

利用 route-map

利用 prefix-list

route-map 不光可以完成过滤，还可以对放行的网络设置策略。

prefix-list 只能完成过滤。

prefix-list 也有默认的 deny all。

prefix-list 的放行所有 ip prefix-list x permit 0.0.0.0/0

利用 正则表达式

BGP 的公认团体属性限制路由更新的发送

路由更新携带 no-advertise，路由更新被接收后不发送给任何 BGP peer。

路由更新携带 no-export 或 local-as，路由更新被接收后不发送给任何 eBGP peer。

CCIE IPSEC VPN

IPSEC VPN

Ip security vpn，是一个工作在网络层的 VPN 技术。

数据在公共网络上面临的风险

  1. 不加密，明文传递
  2. 没有完整性保护，数据容易被篡改
  3. 没有对等体认证，容易受到中间人攻击
  4. 抵赖

IPSEC VPN 解决的问题

  1. 对等体认证，数据发送之前必须经过认证且通过

    口令认证、数字证书（计算机世界的身份证）、加密随机数

  2. 数据加密

    des、3des、aes

  3. 数据完整性保护

    HMAC：哈希消息认证码

  4. 数字签名，防抵赖

    DSA（RSA 公司开发）

KEY：密匙

KEY 是一个工具，可以参与算法作为变量出现，也可以参与认证作为口令出现。

对称密钥、非对称密钥

（加密算法中的 KEY 的理解）

  1. 对等密钥及对称密钥加密算法

    加密解密密钥是同一把

    坏处：不够安全、Key 的共享存在风险

    好处：简单高效

  2. 非对称密钥及非对称密钥算法

    非对称密钥是一对KEY，公钥和密钥。

    公钥最后会公开，密钥不公开。

    公钥加密的数据，只有私钥可以解开。

    私钥加密的数据，只有公钥可以解开。

    坏处：复杂，数据会变庞大

    好处：安全可靠

IPSEC VPN 的加密密钥

非对称密钥共享是用对称密钥及算法保护

数据本身是用非对称密钥及算法保护

密钥产生过程和共享方法

DH 算法负责产生密钥和共享密钥

IPSEC VPN 的关键组件

ISAKMP：互联网安全关联密钥管理协议

  • ISA：通讯双方建立了安全连接就叫做一个 SA 成功了。

    IPSEC VPN 的 SA 有两个：一个是保护 VPN 的，一个是保护数据的。

  • KMP：KMP 是通过 IKE（互联网密钥交换机制）完成的，IKE 专门负责产生密钥，共享密钥。

    IKE 分两个阶段：阶段1产生和共享VPN自身所需的密钥，阶段2利用阶段1的密钥算出新密钥，用于保护用户数据。

IPSEC VPN 配置流程

IOS 中必须带 K9 字样，才支持加密算法： show version

步骤5：配置转换集定义数据加密方式

步骤6：定义加密映射，将数据加密转换集、对等体、感兴趣流绑定

步骤7：应用加密映射（只能用在发送流量的接口）

步骤8：查看命令

查看数据是否被加密show crypto engine connections active

ESP 和 AH

ESP 和 AH 都是 VPN 头部信息的缩写

帧头、包头、段头、数据、帧尾

数据经历 VPN 保护后变化成如下：

帧头、VPN 头、包头、段头、数据、帧尾

VPN 头就分两种：

  1. ESP：封装安全静荷

    该头负责保护数据的完整性，同时加密数据

  2. AH：认证头

    该头只负责保护数据完整性，不加密数据

IPSEC VPN 的模式

  1. 隧道模式（tunnel）

    VPN 网关通过一个隧道模式的 VPN 连接，保护身后所有的通讯数据。

  2. 传输模式（transport）

    通讯双方自身建立 VPN 连接，保护通讯数据。

区别：

隧道模式的新包头，目的 IP 和源 IP 是 VPN 的对等体地址。

传输模式的新包头，目的 IP 和源 IP 复制了原有的 IP 地址。

VPN 流量几乎永远是先被 inside 接口接收，所以 NAT 会直接修改 IP 包头，导致 VPN 无法识别感兴趣流。

CCIE IS-IS

IS-IS（中间系统到中间系统）

数据传递方式

  • 面向连接 tcp/ip

  • 非面向连接 clns/clnp 非连接的网络服务(负责控制数据传递过程)/非连接的网络协议(网络地址标识)

    最后，ip获得广泛使用，但是CLNS的理念也得到认可，因此保留下来，叫做UDP。

IS-IS术语

  • IS：中间系统 即 路由器router

  • ES：终端系统 （但是现在已经没有使用clnp标识的设备了）

  • clnp：标识

    NET：网络实体标识（网络+主机）128bit

    clnp：clnp地址 NSAP网络服务应用标识

clnp地址解释

49.0001.0000.0000.0001.00

49.0002.0000.0000.0002.00

49.0101.0000.0000.0001.00

49：AFI地址格式标识 A地址 F格式 I授权标识 数字由官方的表定好。47，48，49用于私网。

0101 01/01 域/区域 （区域包含在域内）这里为两个十六进制数

0000.0000.0001 mac地址

00 服务号（作用类似于端口号）

IS-IS 域、区域{骨干区域，常规区域}

域：AS

区域：0，1，2 ……

路由器角色类型

  • OSPF{角色：DR BDR DR-other，类型：ABR ASBR}
  • IS-IS {角色：Dis(不完整)，类型：L1 L2 L1/L2}

伪节点

  • OSPF：DR{减少邻接关系，提高更新效率}
  • IS-IS：Dis

IS-IS类型

  • L2 骨干区域
  • L1 常规区域
  • L1/L2(默认类型) 连接两个区域

OSPF LSA 特殊区域

LSA：Link-State Advertisement 链路状态广播

DBD：数据库描述报文

LSR：链路状态查询

LSU：链路状态更新

OSPF七种LSA类型

  • LSA1（Router LSA）

    起源：区域内每台设备都能产生LSA1

    内容：节点信息{RID network cost}

    扩散：不允许穿越ABR，也就是区域内扩散。

    路由：区域内路由

  • LSA2（Network LSA）

    起源：只有DR可以产生

    内容：拓扑信息{节点的直连情况，谁是DR}

    扩散：区域内

  • LSA3（Network summary LSA）

    起源：ABR

    内容：区域网络信息

    扩散：整个OSPF AS

    路由：区域间路由

  • LSA4（ASBR summary LSA）

    起源：ASBR所在域的ABR

    内容：ASBR的RID

    扩散：整个OSPF

    路由：外部{OE1，OE2}

  • LSA5（Autonomous system external LSA）

    起源：ASBR

    内容：外部路由

    扩散：整个OSPF

    路由：外部{OE1，OE2}

OSPF LSA 特区

LSA6 MOSPF专用（废除）

LSA7 NSSA

LSA8 LSA9 ：OSPFv3

LSA10 口袋路由

LSA11 标记

IS-IS路由类型

L2路由 区域间、外部

L1路由 区域内、内部

  • L1设备上做汇总

VPN

虚拟的私有网络，通过组建虚拟的专网实现数据的安全传递。

VPN的分类

OSI 参考模型分类

  • 应用层VPN：开源的第三方VPN软件（open vpn）
  • 传输层VPN：SSL VPN（安全套字节层 VPN）主要用于HTTP的安全加固和防护。
  • 网络层VPN：IPSEC VPN（ip安全VPN）、DM-VPN（动态多点VPN，思科私有）
  • 数据链路层VPN：L2TP（层2隧道VPN）、PPTP（点到点隧道VPN）微软思科联合开发
  • 物理层VPN：专线

实际应用分类

  1. site to site：站点到站点：L2L（lan to lan）

    IPSEC VPN、DM-VPN、MPLS VPN

  2. remote vpn：远程接入VPN

    L2TP、PPTP、SSL VPN、EZ-VPN（思科私有）、anyconnect VPN（任连VPN）

MPLS VPN

MPLS VPN是在BGP peer之间建立（peer to peer VPN），利用了BGP的对等体共享了企业内网路由信息，构建专有通路，实现安全的通讯。

MPLS VPN术语

  • P-NETWORK：运营商核心网络

  • PE 设备：运营商边界设备

  • P 设备：运营商核心设备

  • C-NETWORK：客户网络

  • CE 设备：客户网络边界路由器

  • VRF：虚拟路由转发表

    路由器拥有一个全局路由表和若干个VRF，每个VRF对应到一个C-NETWORK。

    作用：存储不同 C-NETWORK 的路由，保障这些路由不影响全局路由表，而且不会冲突，且相互独立。

    • VRF 的 RD 值：路由区分值

      RD 值书写格式：XX:YY

      不同 VRF 的路由进入BGP 后，利用 RD值进行区别。

      MPLS VPN peer 之间建立的是 MP-BGP peer 关系，MP-BGP 提供专门的转发表，存储来自不同 VRF的路由。

    • VRF 的 RT 值：路由目标值

      RT 值书写格式：XX:YY

      RT 值分为导入和导出两个方向：

      导入值是用来确认是否学习远端的路由

      导出值是路由被共享时携带

      导入和导出值可以是多个

  • MP-BGP VPNv4 更新：即 MP-BGP 的路由更新

MPLS VPN 的双标签

栈底标签：PE 设备上 MP-BGP 给予的，用来帮助数据寻找正确的 VRF路由

栈顶标签：MPLS LDP 给予的，用来穿越P-NETWORK。

安全加密

GET-VPN（隧道加密传输VPN，思科私有）专门在MPLS VPN 基础上加密数据。

MPLS VPN 排错

C-NETWORK 之间无法正常通讯

  1. 当前设备是否学习对端 C-NETWORK 路由

    • CE 是否将路由共享给当前设备，导致当前设备缺乏必要路由。

      动态路由协议邻接关系没建立，错误的路由更新过滤策略。

    • CE 与 PE 间是否正常交换路由更新

      错误的邻接关系、PE 和 CE 间为满足 MPLS VPN 对 OSPF 的要求

      PE 上忘记把 BGP 重分发进底层 VRF

    • PE 设备没有从对端 PE 学习路由

      show ip bgp vpnv4 all 查看MPBGP 转发表，找对端的路由条目。

      MP-BGP 的对等体关系有问题

      show ip bgp vpnv4 all summary 查看对等体关系：

      忘记指定对等体、忘记设置更新源、忘记激活 MP-BGP peer 关系，peer 地址不可达，错误的安全策略导致 BGP TCP 连接无法建立。

      show ip vrf 当前设备缺乏 vrf 的配置

      AS内缺乏路由反射器的配置

    • 对端 PE 设备无法从 CE 设备获取路由信息

  2. 标签问题

      出现了不必要的 NO LABEL，没有远端 LDP 邻居共享标签（当前设备是否缺乏必要的 LDP 邻居，对方没有学习目标网络路由）

跨域MPLS VPN 解决方案详解

方法一：option A：背靠背：back to back

  1. 步骤1：ASBR 上创建 VRF，设定 RD 和 RT，将直连接口划入 VRF。

    R3、R4 创建 VRF 的原因：

  • 利用 VRF 接收来自路由反射器的路由更新，当前 ASBR 设备学习本 AS 站点路由。

  • 创建基于 VRF 的 EBGP PEER 关系，帮助当前 ASBR 设备学习对端 C-NETWORK 路由，并且装载进 VRF 表。

    注意！！ PE 上启用 VRF，切记要将 VRF 和 BGP 相互做重分发，如果 PE 和 CE 间是 BGP，那么来自 CE 的路由更新直接进 PE 的 MP-BGP 转发表。

  • ASBR 创建 VRF 给对端 C-NETWORK 路由新的栈底标签。

方法一如何传递数据

方法一的特点：ASBR 间只有传统的 EBGP peer，ASBR 间没有 MPLS，ASBR 间数据最后是 IP 形式穿越。

方法二：option B 2a

步骤一：ASBR上关闭 RT 过滤功能，在不创建 VRF 的情况下学习 MP-BGP 路由更新，达到学习各自 AS 内部路由的目的

此时栈底标签也跟着共享过来了

步骤二：ASBR 间利用直连网络建立 MP-BGP 关系

方法三：option B 2b

步骤1：ASBR 自身关闭 RT 筛选

步骤2：ASBR 利用静态路由指明抵达对端 ASBR 的 loopback 0 口地址

步骤3：ASBR 间启用 MPLS

如上步骤是考虑到 ASBR 间可能非直连，存在路由黑洞，所以利用 MPLS 的标签，关联帮助数据穿越黑洞。

步骤4：ASBR 间利用环回口建立 MP-BGP EBGP peer

步骤5：将 ASBR 上的静态路由重分发进 BGP

方法四：option C

步骤1：ASBR 间建立 BGPv4 EBGP peer，并且给对方共享标签信息，对对方设置权重值为1.

步骤2：ASBR 上将 PE 和 RR 的 lo0 宣告进 BGP

如上步骤是为了 ASBR 交换各自 AS 内路由反射器及 PE 的路由信息，

ASBR 一定要对 AS 内设备配置 NEXT-HOP-SELF。

步骤3：两个不同的 AS 利用各自路由反射器 lo0 建立 MP-BGP EBGP peer 关系，针对对端做 next-hop-unchange。

步骤4：ASBR 上把来自于对端 AS 的路由重分发进底层 ISIS，将原本被 BGP 学习的路由带入 ISIS，使得当前 AS 的 MPLS 可以分配栈顶标签。

方法四特点：不同 AS 路由反射器直接建立 MP-BGP 对等体关系，使得不同 AS 的 PE 似乎和对端是直接交换路由更新。

如何共享不同站点的路由

  1. R2 与 R5 建立 MP-BGP peer，共享 VPNv4 路由更新

    R3 R4 利用直连建立了 eBGP peer、R3 R4将 R1 R2 R5 R6 ——————没记

  2. 数据传递时标签的问题：顶部标签、底部标签

    LDP 对本地直连、IGPS 协议所学习的路由分配标签

    LDP 无法对 BGP 路由分配标签
