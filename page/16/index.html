CCIE STP生成树协议

STP生成树协议 7.27

一个广播域中选举一个生成树。

广播风暴：绝不停止。

重复帧。

mac地址表的不稳定。

1一个网络中选取一个根桥

2根桥上所有端口都为指定端口

3非根桥上选取一个根端口

4每一段选举一个指定端口

5非指定端口为阻塞端口

*粗体为生成树定义的交换机接口角色

根端口：距离根桥距离最短的端口。开销值一样的话就选最小的发送者id。只在非桥根上选举，并且只选一个。

指定端口：可以执行数据的转发。一台交换机上可以有多个指定端口。根桥上所有端口都是指定端口。

阻塞端口：逻辑上的阻塞，不执行数据的转发，但会持续侦听bpdu信息。

BPDU(桥协议数据单元)只允许根桥产生

root id

优先级：(默认32768，越小越优，更改 优先值时必须为4096的倍数)。

背板mac：标识一台交换机，背板mac小的为优。

bridge id

优先级+背板mac：和根id一样

cost

开销值

port id

优先级(默认128，小优，更改时必须为64的倍数)

端口mac：发送者的端口mac

PVST：每vlan生成树，在优先级插入vlan的信息。优先级+vlan编号。

端口状态

Blocking（阻塞状态）：此时，二层端口为非指定端口，也不会参与数据帧的转发。该端口通过接收BPDU来判断根交换机的位置和根ID，以及在STP拓扑收敛结束之后，各交换机端口应该处于什么状态，在默认情况下，端口会在这种状态下停留20秒钟时间。

Listening（侦听状态）：生成树此时已经根据交换机所接收到的BPDU而判断出了这个端口应该参与数据帧的转发。于是交换机端口就将不再满足于接收BPDU，而同时也开始发送自己的BPDU，并以此通告邻接的交换机该端口会在活动拓扑中参与转发数据帧的工作。在默认情况下，该端口会在这种状态下停留15秒钟的时间。

Learning(学习状态)：这个二层端口准备参与数据帧的转发，并开始填写MAC表。在默认情况下，端口会在这种状态下停留15秒钟时间。

Forwarding（转发状态）：这个二层端口已经成为了活动拓扑的一个组成部分，它会转发数据帧，并同时收发BPDU。

Disabled（禁用状态）：这个二层端口不会参与生成树，也不会转发。

CCIE NAT 地址转换协议

NAT 地址转换协议

优缺点

优点
在地址重叠时提供解决方案

在网络发生变化时避免重新编织

最大作用为可将私网地址转换为公网地址，提升了ip地址空间的使用率，这也是ipv4至今还未耗尽的原因

缺点

增加交换延迟

无法进行端到端ip跟踪

有些应用程序无法正常运行

网络地址转换类型

静态NAT：在本地地址和全局地址直接进行一对一的映射。必须为网络中的每台主机提供一个公有ip地址。

动态NAT：将未注册的ip地址映射到注册ip地址池中的一个地址。必须有足够多的公有ip地址，让连接到因特网的主机都能够同时发送和接收分组。

NAT重载：最常用的NAT类型

也被称为端口地址转换(PAT)

利用源端口将多个非注册ip地址映射到一个注册ip地址(多对一)

使用传输层端口号来标识主机，因此理论上最多可让大约65 000台主机共用一个公有ip地址

本地地址：进行NAT转换前的地址

内部本地地址：转换前的内部源地址，主机的私网地址

外部本地地址：转换后的目标地址，与ISP相连的路由器接口的地址，通常为共有地址

全局地址：进行NAT转换后的地址，通常为因特网共有地址

内部全局地址：转换后的源地址

外部全局地址：转换前的外部目标地址

配置方式

接口角色：
inside：连接在内部广播域
outside：连接公网

转换接口角色
int xx
ip nat inside/outside

静态NAT

ip nat inside source static 内部ip 外部ip

动态NAT

ip nat pool todd 170.168.2.3 170.168.2.254 netmask 255.255.255.0

创建一个地址池todd，用于将全局地址分配给主机

ip nat inside source list 1 pool todd

将与access-list 1匹配的ip地址转换为ip地址池todd中的一个可用地址

PAT(NAT重载)

与动态NAT相比，PAT地址池只包含一个ip地址，且配置命令末尾包含关键字overload

ip nat pool globalnet 170.168.2.1170.168.2.1 netmask 255.255.255.0

ip nat inside source list 1 pool overload

(重复使用，如果地址池有其他地址，也可以使用它，适合大型网络，必须重载多个公有ip地址)

验证与修改

show ip nat translation

debug ip nat

show ip nat statistics

命令claer ip nat translations 清除转换条目，可以使用全局地址、本地地址、tcp(udp)端口、*（星号，清除所有转换条目）

只要NAT表中有包含地址池中地址的转换条目，思科ios就不允许修改或删除相应的地址池

转换条目超时时间：默认为24小时，可以使用命令ip nat translation timeout修改。如果定时器到期，相应的转换条目将从NAT表中删除，动态分配给它的地址将归还给地址池。

步骤4：完成 spoke 上 NHRP 的配置

步骤6：完成 IPSEC 的配置并且套用到隧道上

IS-IS（中间系统到中间系统）

数据传递方式

  • 面向连接 tcp/ip

  • 非面向连接 clns/clnp 非连接的网络服务(负责控制数据传递过程)/非连接的网络协议(网络地址标识)

    最后，ip获得广泛使用，但是CLNS的理念也得到认可，因此保留下来，叫做UDP。

IS-IS术语

  • IS：中间系统 即 路由器router

  • ES：终端系统 （但是现在已经没有使用clnp标识的设备了）

  • clnp：标识

    NET：网络实体标识（网络+主机）128bit

    clnp：clnp地址 NSAP网络服务应用标识

clnp地址解释

49.0001.0000.0000.0001.00

49.0002.0000.0000.0002.00

49.0101.0000.0000.0001.00

49：AFI地址格式标识 A地址 F格式 I授权标识 数字由官方的表定好。47，48，49用于私网。

0101 01/01 域/区域 （区域包含在域内）这里为两个十六进制数

0000.0000.0001 mac地址

00 服务号（作用类似于端口号）

IS-IS 域、区域{骨干区域，常规区域}

域：AS

区域：0，1，2 ……

路由器角色类型

  • OSPF{角色：DR BDR DR-other，类型：ABR ASBR}
  • IS-IS {角色：Dis(不完整)，类型：L1 L2 L1/L2}

伪节点

  • OSPF：DR{减少邻接关系，提高更新效率}
  • IS-IS：Dis

  • L2 骨干区域
  • L1 常规区域
  • L1/L2(默认类型) 连接两个区域

OSPF LSA 特殊区域

LSA：Link-State Advertisement 链路状态广播

DBD：数据库描述报文

LSR：链路状态查询

LSU：链路状态更新

OSPF七种LSA类型

  • LSA1（Router LSA）

    起源：区域内每台设备都能产生LSA1

    内容：节点信息{RID network cost}

    扩散：不允许穿越ABR，也就是区域内扩散。

    路由：区域内路由

  • LSA2（Network LSA）

    起源：只有DR可以产生

    内容：拓扑信息{节点的直连情况，谁是DR}

    扩散：区域内

  • LSA3（Network summary LSA）

    起源：ABR

    内容：区域网络信息

    扩散：整个OSPF AS

    路由：区域间路由

  • LSA4（ASBR summary LSA）

    起源：ASBR所在域的ABR

    内容：ASBR的RID

    扩散：整个OSPF

    路由：外部{OE1，OE2}

  • LSA5（Autonomous system external LSA）

    起源：ASBR

    内容：外部路由

    扩散：整个OSPF

    路由：外部{OE1，OE2}

CCIE IPSEC VPN

IPSEC VPN

Ip security vpn，是一个工作在网络层的 VPN 技术。

数据在公共网络上面临的风险

  1. 不加密，明文传递
  2. 没有完整性保护，数据容易被篡改
  3. 没有对等体认证，容易受到中间人攻击
  4. 抵赖

IPSEC VPN 解决的问题

  1. 对等体认证，数据发送之前必须经过认证且通过

    口令认证、数字证书（计算机世界的身份证）、加密随机数

  2. 数据加密

    des、3des、aes

  3. 数据完整性保护

    HMAC：哈希消息认证码

  4. 数字签名，防抵赖

    DSA（RSA 公司开发）

KEY：密匙

KEY 是一个工具，可以参与算法作为变量出现，也可以参与认证作为口令出现。

对称密钥、非对称密钥

（加密算法中的 KEY 的理解）

  1. 对等密钥及对称密钥加密算法

    加密解密密钥是同一把

    坏处：不够安全、Key 的共享存在风险

    好处：简单高效

  2. 非对称密钥及非对称密钥算法

    非对称密钥是一对KEY，公钥和密钥。

    公钥最后会公开，密钥不公开。

    公钥加密的数据，只有私钥可以解开。

    私钥加密的数据，只有公钥可以解开。

    坏处：复杂，数据会变庞大

    好处：安全可靠

IPSEC VPN 的加密密钥

非对称密钥共享是用对称密钥及算法保护

数据本身是用非对称密钥及算法保护

密钥产生过程和共享方法

DH 算法负责产生密钥和共享密钥

IPSEC VPN 的关键组件

ISAKMP：互联网安全关联密钥管理协议

  • ISA：通讯双方建立了安全连接就叫做一个 SA 成功了。

    IPSEC VPN 的 SA 有两个：一个是保护 VPN 的，一个是保护数据的。

  • KMP：KMP 是通过 IKE（互联网密钥交换机制）完成的，IKE 专门负责产生密钥，共享密钥。

    IKE 分两个阶段：阶段1产生和共享VPN自身所需的密钥，阶段2利用阶段1的密钥算出新密钥，用于保护用户数据。

ESP 和 AH

ESP 和 AH 都是 VPN 头部信息的缩写

帧头、包头、段头、数据、帧尾

数据经历 VPN 保护后变化成如下：

帧头、VPN 头、包头、段头、数据、帧尾

VPN 头就分两种：

  1. ESP：封装安全静荷

    该头负责保护数据的完整性，同时加密数据

  2. AH：认证头

    该头只负责保护数据完整性，不加密数据

IPSEC VPN 的模式

  1. 隧道模式（tunnel）

    VPN 网关通过一个隧道模式的 VPN 连接，保护身后所有的通讯数据。

  2. 传输模式（transport）

    通讯双方自身建立 VPN 连接，保护通讯数据。

区别：

隧道模式的新包头，目的 IP 和源 IP 是 VPN 的对等体地址。

传输模式的新包头，目的 IP 和源 IP 复制了原有的 IP 地址。

access-list命令

（1）标准访问列表

access-list access-list-number { permit | deny } {source [source-wildcard] | any}

命令解释如下。

access-list：访问列表命令。

access-list-number：访问列表号码，值为1~99.

permit：允许。

deny：拒绝。

source：源IP地址。

source-wildcard：源IP地址的通配符。

（2）扩展访问列表

access-list access-list-number { permit | deny } { protocol \ protocol-keyword } { source [ source-wildcard ] | any } { destination destination-wildcard } | any }[protocol-specific options][log]

命令解释如下。

access-list-number：访问列表号码，值为100~199.

protocol \ protocol-keyword：可使用的协议，包括IP、ICMP、IGRP、EIGRP、OSPF等。

destination destination-wild：目的IP地址，格式与源IP地址相同。

protocol-specific options：协议制定的选项。

log：记录有关数据报进入访问列表的信息。

rip 跳数

offset-list acc号 in/out 跳数增加值
