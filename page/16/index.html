CCIE 防火墙

防火墙

防火墙的分类

硬件防火墙：流量的省查及访问的控制：保护的是网络

软件防火墙：省查木马或病毒：保护的是系统

应用防火墙：WAF（WEB 应用防火墙）、APP 防火墙

硬件防火墙

checkpoint（诺基亚）

netscreen（詹博、Juniper）

PIX、ASA（Cisco）SDN（安全自防御网络、软件定义的网络）

华为、天融信

硬件防火墙的部署

网络的边缘部署防火墙，一般放在 PPPOE 拨号设备的身后，关键节点之间。

防火墙的基础知识

防火墙工作模式

路由模式：（默认模式）防火墙读取 IP 包头，进行网络层的保护，支持动态路由协议和静态路由。

交换模式：

防火墙的多模式和单模式

多模式支持虚拟防火墙，即把一台防火墙配置成多台逻辑防火墙。

单模式是默认模式。

防火墙接口的类型

inside：内部接口

通常连接的是信任网络即内网

outside：外部接口

通常连接的是非信任公共网络，比如 ISP 或 其他组织网络

DMZ：非军事区域

通常数据中心放在 DMZ 区域

防火墙接口等级和访问规则

inside 接口安全等级：100

outside 接口安全等级：0

DMZ 接口安全等级：50、0

高等级接口接收流量穿越防火墙从低等级接口发送出去，必须经过 NAT。

低等级接口接收流量穿越防火墙从高等级接口发送出去，默认不允许。

除非是返回流量或被管理员手动放行。

相同安全级别的接口相互通讯，默认不允许通讯，除非管理员放行。

route-map、prefix-list、filter-list（筛选列表）

利用 route-map

可以是 in 方向，也可以是 out 方向。

利用 prefix-list

route-map 不光可以完成过滤，还可以对放行的网络设置策略。

prefix-list 只能完成过滤。

prefix-list 也有默认的 deny all。

prefix-list 的放行所有 ip prefix-list x permit 0.0.0.0/0

CCIE IPSEC VPN

IPSEC VPN

Ip security vpn，是一个工作在网络层的 VPN 技术。

数据在公共网络上面临的风险

  1. 不加密，明文传递
  2. 没有完整性保护，数据容易被篡改
  3. 没有对等体认证，容易受到中间人攻击
  4. 抵赖

IPSEC VPN 解决的问题

  1. 对等体认证，数据发送之前必须经过认证且通过

    口令认证、数字证书（计算机世界的身份证）、加密随机数

  2. 数据加密

    des、3des、aes

  3. 数据完整性保护

    HMAC：哈希消息认证码

  4. 数字签名，防抵赖

    DSA（RSA 公司开发）

KEY：密匙

KEY 是一个工具，可以参与算法作为变量出现，也可以参与认证作为口令出现。

对称密钥、非对称密钥

（加密算法中的 KEY 的理解）

  1. 对等密钥及对称密钥加密算法

    加密解密密钥是同一把

    坏处：不够安全、Key 的共享存在风险

    好处：简单高效

  2. 非对称密钥及非对称密钥算法

    非对称密钥是一对KEY，公钥和密钥。

    公钥最后会公开，密钥不公开。

    公钥加密的数据，只有私钥可以解开。

    私钥加密的数据，只有公钥可以解开。

    坏处：复杂，数据会变庞大

    好处：安全可靠

IPSEC VPN 的加密密钥

非对称密钥共享是用对称密钥及算法保护

数据本身是用非对称密钥及算法保护

密钥产生过程和共享方法

DH 算法负责产生密钥和共享密钥

IPSEC VPN 的关键组件

ISAKMP：互联网安全关联密钥管理协议

  • ISA：通讯双方建立了安全连接就叫做一个 SA 成功了。

    IPSEC VPN 的 SA 有两个：一个是保护 VPN 的，一个是保护数据的。

  • KMP：KMP 是通过 IKE（互联网密钥交换机制）完成的，IKE 专门负责产生密钥，共享密钥。

    IKE 分两个阶段：阶段1产生和共享VPN自身所需的密钥，阶段2利用阶段1的密钥算出新密钥，用于保护用户数据。

ESP 和 AH

ESP 和 AH 都是 VPN 头部信息的缩写

帧头、包头、段头、数据、帧尾

数据经历 VPN 保护后变化成如下：

帧头、VPN 头、包头、段头、数据、帧尾

VPN 头就分两种：

  1. ESP：封装安全静荷

    该头负责保护数据的完整性，同时加密数据

  2. AH：认证头

    该头只负责保护数据完整性，不加密数据

IPSEC VPN 的模式

  1. 隧道模式（tunnel）

    VPN 网关通过一个隧道模式的 VPN 连接，保护身后所有的通讯数据。

  2. 传输模式（transport）

    通讯双方自身建立 VPN 连接，保护通讯数据。

区别：

隧道模式的新包头，目的 IP 和源 IP 是 VPN 的对等体地址。

传输模式的新包头，目的 IP 和源 IP 复制了原有的 IP 地址。

NAT 对 IPSEC VPN 的影响

VPN 流量几乎永远是先被 inside 接口接收，所以 NAT 会直接修改 IP 包头，导致 VPN 无法识别感兴趣流。

交换机指示灯：

绿色常亮：接口两端没有问题，但是当前没有数据从中通过

绿色不停地闪：正在传输数据

橘红色慢闪：端口没问题,对端线路或者对端的端口有问题

橘红色快闪：端口和对端连接没有问题，正在协商参数建立

黄色闪动：物理接口有问题，尝试重启。

不闪： 物理接口有问题，无法通信

vlan（虚拟局域网）

作用：分隔广播域

所有接口默认属于vlan1（默认vlan，不可删除）。

vlan ID范围和解释

\1. 0和4095 仅仅限于系统使用 用户不能查看和使用

\2. 1 正常 cisco默认的vlan,用户可以使用但是不能删除

\3. 2–1001 正常 用于以太网的VLAN，用户可以创建。使用。删除。

\4. 1002–1005 正常 用于FDDI和令牌环的CISCO默认VLAN，用户不能删除

\5. 1006–1024 保留 仅仅限于系统使用。用户不能查看和使用。

\6. 1025–4094 扩展 仅仅用于以太网VLAN

用于同步同一网络中交换机上网络的信息（减小庞大的配置量）

CCIE STP生成树协议

STP生成树协议 7.27

一个广播域中选举一个生成树。

广播风暴：绝不停止。

重复帧。

mac地址表的不稳定。

1一个网络中选取一个根桥

2根桥上所有端口都为指定端口

3非根桥上选取一个根端口

4每一段选举一个指定端口

5非指定端口为阻塞端口

*粗体为生成树定义的交换机接口角色

根端口：距离根桥距离最短的端口。开销值一样的话就选最小的发送者id。只在非桥根上选举，并且只选一个。

指定端口：可以执行数据的转发。一台交换机上可以有多个指定端口。根桥上所有端口都是指定端口。

阻塞端口：逻辑上的阻塞，不执行数据的转发，但会持续侦听bpdu信息。

BPDU(桥协议数据单元)只允许根桥产生

root id

优先级：(默认32768，越小越优，更改 优先值时必须为4096的倍数)。

背板mac：标识一台交换机，背板mac小的为优。

bridge id

优先级+背板mac：和根id一样

cost

开销值

port id

优先级(默认128，小优，更改时必须为64的倍数)

端口mac：发送者的端口mac

PVST：每vlan生成树，在优先级插入vlan的信息。优先级+vlan编号。

端口状态

Blocking（阻塞状态）：此时，二层端口为非指定端口，也不会参与数据帧的转发。该端口通过接收BPDU来判断根交换机的位置和根ID，以及在STP拓扑收敛结束之后，各交换机端口应该处于什么状态，在默认情况下，端口会在这种状态下停留20秒钟时间。

Listening（侦听状态）：生成树此时已经根据交换机所接收到的BPDU而判断出了这个端口应该参与数据帧的转发。于是交换机端口就将不再满足于接收BPDU，而同时也开始发送自己的BPDU，并以此通告邻接的交换机该端口会在活动拓扑中参与转发数据帧的工作。在默认情况下，该端口会在这种状态下停留15秒钟的时间。

Learning(学习状态)：这个二层端口准备参与数据帧的转发，并开始填写MAC表。在默认情况下，端口会在这种状态下停留15秒钟时间。

Forwarding（转发状态）：这个二层端口已经成为了活动拓扑的一个组成部分，它会转发数据帧，并同时收发BPDU。

Disabled（禁用状态）：这个二层端口不会参与生成树，也不会转发。

根桥每2s发送一次bpdu信息，维护本生成树的运行。

tcn bpdu 拓扑变更通告，只会从根端口向上发。

CCIE DM-VPN

DM-VPN

动态多点VPN、Cisco私有

IPSEC VPN 存在的问题

  1. 不支持多播
  2. 复杂的拓扑

DM-VPN 的术语

  • M-GRE（多点隧道）一点对多点
  • 物理 IP 地址（物理接口）/逻辑 IP 地址（tunnel 私有 IP）
  • NHRP（下一跳解析协议、总部和分部都启用）
    NHS（下一跳服务器、总部）

DM-VPN 的三个阶段

阶段一

  • M-GRE：EIGRP
  • 分支和分支要通过总部才能相互通讯

阶段二

  • M-GRE：EIGRP
  • 分支和分支可以直接通讯

阶段三

  • 单云双中心（两个 HUB 同时工作互为备份，还是依赖 DM-VPN 阶段2）

DM-VPN 的基础配置流程

步骤1：完成 HUB 上 M-GRE 的配置

步骤2：完成 spoke 上的 M-GRE 配置

步骤3：完成 HUB 上隧道 NHRP 的配置

步骤4：完成 spoke 上 NHRP 的配置

步骤5：隧道逻辑 IP 地址参与 EIGRP

步骤6：完成 IPSEC 的配置并且套用到隧道上

杂：

查看 HUB 的分支信息：show ip nhrp

关闭 EIGRP 水平分割：no ip split-horizon eigrp 1

开启 NHRP 捷径：ip nhrp shortcat

单云双中心的配置细节

区别1：两个 HUB 需要互做 NHRP 的映射。

区别2：两个 spoke 需要设定两个 NHS 服务器，并且谁先被设定谁就是主设备。

MPLS详解

multiple protocol label switch（多协议标签交换）

MPLS作用

MPLS是一种数据交换机制，基于标签库和数据携带的标签。

进行数据转发，同时支持多种协议。

MPLS在实际工作中，能实现如下两个基本功能：

  1. 数据包的高速转发
  2. 减少参与BGP的设备数量、减少对等体关系、减少不必要的路由学习

MPLS术语

label 标签

总长 32bit 由如下几个部分组成：

前20bit 是标签取值范围

第21bit 是栈底位，栈底位为1则表明该标签已经是最后一个

第22~24bit 是体验位，可以将标签对应的数据分成8个级别，给予不同的传输策略。

最后8bit 是TTL，限制携带标签的数据传递范围。

MPLS 模式

帧模式：frame mode（主流模式），标签被插在帧头的后面，包头前面。

信令模式：cell mode 标签被插在信令头后面。

LSR：标签交换路由器

启用了MPLS的路由器统称LSR。

入站 LSR （inbound LSR）：接收到 IP 数据包，压入标签，传递给 MPLS 网络。

链路中 LSR （link LSR）：接收到携带标签的数据，基于标签进行数据转发。

出站 LSR （outbound LSR）：负责将标签移除，并且变回 IP 数据报文。

压入、交换、移除、弹出

压入：入站LSR将标签压入数据。

交换：链路中LSR基于标签库，进行标签的置换、数据的传递。

移除：出站LSR将数据携带的所有标签全部删除。

弹出：出站LSR将数据携带的栈顶标签弹出，其他标签保留。

LSP：链路标签路径

是MPLS设备传递数据所需的路径。（即路由）

LSP是通过标签的关联产生的。

标签分配、分发、关联、维护协议

TDP：标记分发协议、思科私有

LDP：标签分发协议、公有公开

LDP详解

LDP作用

  1. 利用 hello 消息建立 LDP 邻接关系。
  2. 基于CEF给本地网络分配本地标签。
  3. 将网络对应的标签分发共享给邻居
  4. 基于底层协议完成本地标签与远程共享标签的关联，最终形成LSP。
  5. 负责监控LSP的有效性，并且维护它。

LDP邻接关系的建立

LDP 邻居的发现，通过hello消息，基于UDP，目标端口646。

LDP 利用TCP，目标端口646完成邻接关系建立。

LDP可以设定RID，RID扮演更新源，更新源会成为对端TCP的目标接收地址，所以更新源必须可达。

LDP标签的分配

除BGP路由，LDP可以给本地直连网络以及非直连网络分配标签。

LDP标签的共享

两种形式：下游主动、上游主动。

实际上标签的分享是给所有的邻居，不遵循所谓的上下游。

LDP标签的关联

LDP永远将路由路径下一跳共享的标签与对应网络的本地标签关联。

boundaddress：绑定地址，参与MPLS设备的IP会成为绑定地址和，会跟着本地标签一起共享给LDP邻居，邻居根据底层协议下一跳地址和绑定地址的判断来确认哪个LDP邻居是下一跳，再绑定。

LDP的特殊标签

  • UNTAGE：未标记

出现原因：底层是ospf,本地网络无法找到对应的远程标签，LDP给网络分配untage。

处理方式：目标网络远程标签如果是untage，那么LSR会将所有标签移除，向外传递。

  • POP：倒数第二跳标签

出现原因：当前路由器给本地直连网络分配的本地标签就是POP。

处理方式：如果出站标签是POP，那么LSR 会将栈顶标签弹出。

意义：提高数据转发效率，减少一次标签库查找。

LDP的两张表

LIB：标签信息库

目标网络本地标签和学习到的远程标签存储在这里。（未关联）

LFIB：标签转发信息库

标签关联后的结果 即 LSP 存储在这里。

MPLS的配置流程

步骤1：开启 CEF（默认开启，一般情况不需要此命令）

ip cef

步骤2：设置标签分配、分发关联协议（默认是LDP）

mpls label protocol <ldp/tdp>

步骤3：设置标签取值范围（通常不设置）

mpls label range <xx-yy>

步骤4：设置 LDP 的 RID 即设置更新源（不设置则和OSPF一样自动选取RID）

mpls ldp router-id loopback x [force] //RID 必须可达、force 强制改变 RID

步骤5：接口下启用MPLS

mpls ip
