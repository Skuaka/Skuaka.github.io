CCIE IPSEC VPN

IPSEC VPN

Ip security vpn，是一个工作在网络层的 VPN 技术。

数据在公共网络上面临的风险

  1. 不加密，明文传递
  2. 没有完整性保护，数据容易被篡改
  3. 没有对等体认证，容易受到中间人攻击
  4. 抵赖

IPSEC VPN 解决的问题

  1. 对等体认证，数据发送之前必须经过认证且通过

    口令认证、数字证书（计算机世界的身份证）、加密随机数

  2. 数据加密

    des、3des、aes

  3. 数据完整性保护

    HMAC：哈希消息认证码

  4. 数字签名，防抵赖

    DSA（RSA 公司开发）

KEY：密匙

KEY 是一个工具，可以参与算法作为变量出现，也可以参与认证作为口令出现。

对称密钥、非对称密钥

（加密算法中的 KEY 的理解）

  1. 对等密钥及对称密钥加密算法

    加密解密密钥是同一把

    坏处：不够安全、Key 的共享存在风险

    好处：简单高效

  2. 非对称密钥及非对称密钥算法

    非对称密钥是一对KEY，公钥和密钥。

    公钥最后会公开，密钥不公开。

    公钥加密的数据，只有私钥可以解开。

    私钥加密的数据，只有公钥可以解开。

    坏处：复杂，数据会变庞大

    好处：安全可靠

IPSEC VPN 的加密密钥

非对称密钥共享是用对称密钥及算法保护

数据本身是用非对称密钥及算法保护

密钥产生过程和共享方法

DH 算法负责产生密钥和共享密钥

IPSEC VPN 的关键组件

ISAKMP：互联网安全关联密钥管理协议

  • ISA：通讯双方建立了安全连接就叫做一个 SA 成功了。

    IPSEC VPN 的 SA 有两个：一个是保护 VPN 的，一个是保护数据的。

  • KMP：KMP 是通过 IKE（互联网密钥交换机制）完成的，IKE 专门负责产生密钥，共享密钥。

    IKE 分两个阶段：阶段1产生和共享VPN自身所需的密钥，阶段2利用阶段1的密钥算出新密钥，用于保护用户数据。

ESP 和 AH

ESP 和 AH 都是 VPN 头部信息的缩写

帧头、包头、段头、数据、帧尾

数据经历 VPN 保护后变化成如下：

帧头、VPN 头、包头、段头、数据、帧尾

VPN 头就分两种：

  1. ESP：封装安全静荷

    该头负责保护数据的完整性，同时加密数据

  2. AH：认证头

    该头只负责保护数据完整性，不加密数据

IPSEC VPN 的模式

  1. 隧道模式（tunnel）

    VPN 网关通过一个隧道模式的 VPN 连接，保护身后所有的通讯数据。

  2. 传输模式（transport）

    通讯双方自身建立 VPN 连接，保护通讯数据。

区别：

隧道模式的新包头，目的 IP 和源 IP 是 VPN 的对等体地址。

传输模式的新包头，目的 IP 和源 IP 复制了原有的 IP 地址。

NAT 对 IPSEC VPN 的影响

VPN 流量几乎永远是先被 inside 接口接收，所以 NAT 会直接修改 IP 包头，导致 VPN 无法识别感兴趣流。

CCIE DM-VPN

DM-VPN

动态多点VPN、Cisco私有

IPSEC VPN 存在的问题

  1. 不支持多播
  2. 复杂的拓扑

DM-VPN 的术语

  • M-GRE（多点隧道）一点对多点
  • 物理 IP 地址（物理接口）/逻辑 IP 地址（tunnel 私有 IP）
  • NHRP（下一跳解析协议、总部和分部都启用）
    NHS（下一跳服务器、总部）

DM-VPN 的三个阶段

阶段一

  • M-GRE：EIGRP
  • 分支和分支要通过总部才能相互通讯

阶段二

  • M-GRE：EIGRP
  • 分支和分支可以直接通讯

阶段三

  • 单云双中心（两个 HUB 同时工作互为备份，还是依赖 DM-VPN 阶段2）

DM-VPN 的基础配置流程

步骤1：完成 HUB 上 M-GRE 的配置

步骤2：完成 spoke 上的 M-GRE 配置

步骤3：完成 HUB 上隧道 NHRP 的配置

步骤4：完成 spoke 上 NHRP 的配置

步骤5：隧道逻辑 IP 地址参与 EIGRP

步骤6：完成 IPSEC 的配置并且套用到隧道上

杂：

查看 HUB 的分支信息：show ip nhrp

关闭 EIGRP 水平分割：no ip split-horizon eigrp 1

开启 NHRP 捷径：ip nhrp shortcat

单云双中心的配置细节

区别1：两个 HUB 需要互做 NHRP 的映射。

区别2：两个 spoke 需要设定两个 NHS 服务器，并且谁先被设定谁就是主设备。

防火墙

防火墙的基础知识

防火墙工作模式

路由模式：（默认模式）防火墙读取 IP 包头，进行网络层的保护，支持动态路由协议和静态路由。

交换模式：

防火墙的多模式和单模式

多模式支持虚拟防火墙，即把一台防火墙配置成多台逻辑防火墙。

单模式是默认模式。

防火墙接口的类型

inside：内部接口

通常连接的是信任网络即内网

outside：外部接口

通常连接的是非信任公共网络，比如 ISP 或 其他组织网络

DMZ：非军事区域

通常数据中心放在 DMZ 区域

防火墙接口等级和访问规则

inside 接口安全等级：100

outside 接口安全等级：0

DMZ 接口安全等级：50、0

高等级接口接收流量穿越防火墙从低等级接口发送出去，必须经过 NAT。

低等级接口接收流量穿越防火墙从高等级接口发送出去，默认不允许。

除非是返回流量或被管理员手动放行。

相同安全级别的接口相互通讯，默认不允许通讯，除非管理员放行。

CCIE NAT 地址转换协议

NAT 地址转换协议

优缺点

优点
在地址重叠时提供解决方案

在网络发生变化时避免重新编织

最大作用为可将私网地址转换为公网地址，提升了ip地址空间的使用率，这也是ipv4至今还未耗尽的原因

缺点

增加交换延迟

无法进行端到端ip跟踪

有些应用程序无法正常运行

网络地址转换类型

静态NAT：在本地地址和全局地址直接进行一对一的映射。必须为网络中的每台主机提供一个公有ip地址。

动态NAT：将未注册的ip地址映射到注册ip地址池中的一个地址。必须有足够多的公有ip地址，让连接到因特网的主机都能够同时发送和接收分组。

NAT重载：最常用的NAT类型

也被称为端口地址转换(PAT)

利用源端口将多个非注册ip地址映射到一个注册ip地址(多对一)

使用传输层端口号来标识主机，因此理论上最多可让大约65 000台主机共用一个公有ip地址

本地地址：进行NAT转换前的地址

内部本地地址：转换前的内部源地址，主机的私网地址

外部本地地址：转换后的目标地址，与ISP相连的路由器接口的地址，通常为共有地址

全局地址：进行NAT转换后的地址，通常为因特网共有地址

内部全局地址：转换后的源地址

外部全局地址：转换前的外部目标地址

配置方式

接口角色：
inside：连接在内部广播域
outside：连接公网

转换接口角色
int xx
ip nat inside/outside

静态NAT

ip nat inside source static 内部ip 外部ip

动态NAT

ip nat pool todd 170.168.2.3 170.168.2.254 netmask 255.255.255.0

创建一个地址池todd，用于将全局地址分配给主机

ip nat inside source list 1 pool todd

将与access-list 1匹配的ip地址转换为ip地址池todd中的一个可用地址

PAT(NAT重载)

与动态NAT相比，PAT地址池只包含一个ip地址，且配置命令末尾包含关键字overload

ip nat pool globalnet 170.168.2.1170.168.2.1 netmask 255.255.255.0

ip nat inside source list 1 pool overload

(重复使用，如果地址池有其他地址，也可以使用它，适合大型网络，必须重载多个公有ip地址)

验证与修改

show ip nat translation

debug ip nat

show ip nat statistics

命令claer ip nat translations 清除转换条目，可以使用全局地址、本地地址、tcp(udp)端口、*（星号，清除所有转换条目）

只要NAT表中有包含地址池中地址的转换条目，思科ios就不允许修改或删除相应的地址池

转换条目超时时间：默认为24小时，可以使用命令ip nat translation timeout修改。如果定时器到期，相应的转换条目将从NAT表中删除，动态分配给它的地址将归还给地址池。

access-list命令

（1）标准访问列表

access-list access-list-number { permit | deny } {source [source-wildcard] | any}

命令解释如下。

access-list：访问列表命令。

access-list-number：访问列表号码，值为1~99.

permit：允许。

deny：拒绝。

source：源IP地址。

source-wildcard：源IP地址的通配符。

（2）扩展访问列表

access-list access-list-number { permit | deny } { protocol \ protocol-keyword } { source [ source-wildcard ] | any } { destination destination-wildcard } | any }[protocol-specific options][log]

命令解释如下。

access-list-number：访问列表号码，值为100~199.

protocol \ protocol-keyword：可使用的协议，包括IP、ICMP、IGRP、EIGRP、OSPF等。

destination destination-wild：目的IP地址，格式与源IP地址相同。

protocol-specific options：协议制定的选项。

log：记录有关数据报进入访问列表的信息。

CCIE 跨域的多播

跨域的多播

源和组成员在不同的AS，需要传递多播数据流。

跨域多播的问题和挑战

组成员无法发现源、源无法发现组成员、多播树无法建立。

MSDP 多播源发现协议

MSDP 是配置在不同 AS RP 上的，它会建立基于 TCP 的对等体关系，利用自身的 SA（源关联）消息共享”源地址信息、多播组地址信息、RP 信息”给对端 RP。

MSDP 配置流程

步骤1：RP上配置新的 loopback 接口，并且宣告进 BGP。

步骤2：RP上完成 MSDP 的配置

排错手段

ip 路由疑难解析

多播排错

ping 多播组地址 不同

  1. 确认发现组成员

    show ip igmp group

    查看组地址、本地接口、报告者 是否出现

    错点

    • 没有模拟成组成员，缺乏接口下：ip igmp join-group <a.b.c.d>
    • 路由器接口没有加入PIM，缺乏接口下:ip pim <x>
    • 连通性问题

  2. 确认稀疏模式获取 RP 信息

    show ip pim rp mapping 查看 RP 映射，查看 RP 的结果

    show ip pim rp 查看多播组和 RP 的关联

    错点

    • 没有通过静态 RP 指明 RP 地址
    • BSR 的 C-BSR 或 C-RP 配置问题
    • auto-rp 的配置问题
    • 缺乏auto-rp 侦听的配置
    • 设备间缺乏必要的PIM 邻居：show ip pim neighbor
      • 设备缺乏多播路由功能和接口忘记加入PIM
      • 接口忘记激活、错误的安全策略挡住了PIM消息
    • 设备无法正常抵达 RP 地址，所以不显示获取的 RP 消息
      • 路由层面问题、忘记宣告、忘记重分发、错误的过滤

  3. RPF 检测未通过

    • 缺乏必要的更新源路由
    • 忘记宣告、忘记重分发、错误的路由更新过滤

  4. 是否是跨域多播、缺乏必要的配置和连通性

    • 缺乏必要的 MSDP 配置和 MSDP 对等体关系

      show run | sec ip msdp

      show ip msdp summary

      MSDP peer 地址不可达

      MSDP 链接源和起源 ID 设置错误

MPLS VPN

MPLS VPN是在BGP peer之间建立（peer to peer VPN），利用了BGP的对等体共享了企业内网路由信息，构建专有通路，实现安全的通讯。

MPLS VPN术语

  • P-NETWORK：运营商核心网络

  • PE 设备：运营商边界设备

  • P 设备：运营商核心设备

  • C-NETWORK：客户网络

  • CE 设备：客户网络边界路由器

  • VRF：虚拟路由转发表

    路由器拥有一个全局路由表和若干个VRF，每个VRF对应到一个C-NETWORK。

    作用：存储不同 C-NETWORK 的路由，保障这些路由不影响全局路由表，而且不会冲突，且相互独立。

    • VRF 的 RD 值：路由区分值

      RD 值书写格式：XX:YY

      不同 VRF 的路由进入BGP 后，利用 RD值进行区别。

      MPLS VPN peer 之间建立的是 MP-BGP peer 关系，MP-BGP 提供专门的转发表，存储来自不同 VRF的路由。

    • VRF 的 RT 值：路由目标值

      RT 值书写格式：XX:YY

      RT 值分为导入和导出两个方向：

      导入值是用来确认是否学习远端的路由

      导出值是路由被共享时携带

      导入和导出值可以是多个

  • MP-BGP VPNv4 更新：即 MP-BGP 的路由更新

MPLS VPN 的双标签

栈底标签：PE 设备上 MP-BGP 给予的，用来帮助数据寻找正确的 VRF路由

栈顶标签：MPLS LDP 给予的，用来穿越P-NETWORK。

安全加密

GET-VPN（隧道加密传输VPN，思科私有）专门在MPLS VPN 基础上加密数据。

MPLS VPN 排错

C-NETWORK 之间无法正常通讯

  1. 当前设备是否学习对端 C-NETWORK 路由

    • CE 是否将路由共享给当前设备，导致当前设备缺乏必要路由。

      动态路由协议邻接关系没建立，错误的路由更新过滤策略。

    • CE 与 PE 间是否正常交换路由更新

      错误的邻接关系、PE 和 CE 间为满足 MPLS VPN 对 OSPF 的要求

      PE 上忘记把 BGP 重分发进底层 VRF

    • PE 设备没有从对端 PE 学习路由

      show ip bgp vpnv4 all 查看MPBGP 转发表，找对端的路由条目。

      MP-BGP 的对等体关系有问题

      show ip bgp vpnv4 all summary 查看对等体关系：

      忘记指定对等体、忘记设置更新源、忘记激活 MP-BGP peer 关系，peer 地址不可达，错误的安全策略导致 BGP TCP 连接无法建立。

      show ip vrf 当前设备缺乏 vrf 的配置

      AS内缺乏路由反射器的配置

    • 对端 PE 设备无法从 CE 设备获取路由信息

  2. 标签问题

    PE 上查看 BGP 路由的下一跳，确认栈顶标签关联的目标。

    show ip bgp vpnv4 all 通过该命令查看下一跳

    错点

    • 标签路径不关联

      出现了不必要的 UNTAGE，缺乏 loopback 下 ip OSPF network point-to-point

      出现了不必要的 NO LABEL，没有远端 LDP 邻居共享标签（当前设备是否缺乏必要的 LDP 邻居，对方没有学习目标网络路由）

  3. ping 包源对方是否可达

  4. 错误的访问控制策略

跨域MPLS VPN 解决方案详解

方法一：option A：背靠背：back to back

  1. 步骤1：ASBR 上创建 VRF，设定 RD 和 RT，将直连接口划入 VRF。

    R3、R4 创建 VRF 的原因：

  • 利用 VRF 接收来自路由反射器的路由更新，当前 ASBR 设备学习本 AS 站点路由。

  • 创建基于 VRF 的 EBGP PEER 关系，帮助当前 ASBR 设备学习对端 C-NETWORK 路由，并且装载进 VRF 表。

    注意！！ PE 上启用 VRF，切记要将 VRF 和 BGP 相互做重分发，如果 PE 和 CE 间是 BGP，那么来自 CE 的路由更新直接进 PE 的 MP-BGP 转发表。

  • ASBR 创建 VRF 给对端 C-NETWORK 路由新的栈底标签。

方法一如何传递数据

方法一的特点：ASBR 间只有传统的 EBGP peer，ASBR 间没有 MPLS，ASBR 间数据最后是 IP 形式穿越。

方法二：option B 2a

步骤一：ASBR上关闭 RT 过滤功能，在不创建 VRF 的情况下学习 MP-BGP 路由更新，达到学习各自 AS 内部路由的目的

此时栈底标签也跟着共享过来了

步骤二：ASBR 间利用直连网络建立 MP-BGP 关系

方法三：option B 2b

步骤1：ASBR 自身关闭 RT 筛选

步骤2：ASBR 利用静态路由指明抵达对端 ASBR 的 loopback 0 口地址

步骤3：ASBR 间启用 MPLS

如上步骤是考虑到 ASBR 间可能非直连，存在路由黑洞，所以利用 MPLS 的标签，关联帮助数据穿越黑洞。

步骤4：ASBR 间利用环回口建立 MP-BGP EBGP peer

步骤5：将 ASBR 上的静态路由重分发进 BGP

方法四：option C

步骤1：ASBR 间建立 BGPv4 EBGP peer，并且给对方共享标签信息，对对方设置权重值为1.

步骤2：ASBR 上将 PE 和 RR 的 lo0 宣告进 BGP

如上步骤是为了 ASBR 交换各自 AS 内路由反射器及 PE 的路由信息，

ASBR 一定要对 AS 内设备配置 NEXT-HOP-SELF。

步骤3：两个不同的 AS 利用各自路由反射器 lo0 建立 MP-BGP EBGP peer 关系，针对对端做 next-hop-unchange。

步骤4：ASBR 上把来自于对端 AS 的路由重分发进底层 ISIS，将原本被 BGP 学习的路由带入 ISIS，使得当前 AS 的 MPLS 可以分配栈顶标签。

方法四特点：不同 AS 路由反射器直接建立 MP-BGP 对等体关系，使得不同 AS 的 PE 似乎和对端是直接交换路由更新。

如何共享不同站点的路由

  1. R2 与 R5 建立 MP-BGP peer，共享 VPNv4 路由更新

    R3 R4 利用直连建立了 eBGP peer、R3 R4将 R1 R2 R5 R6 ——————没记

  2. 数据传递时标签的问题：顶部标签、底部标签

    LDP 对本地直连、IGPS 协议所学习的路由分配标签

    LDP 无法对 BGP 路由分配标签

边界网关路由协议

BGP简介

工作类型：路径矢量路由选择协议

基于tcp(三次握手)

三张表：邻居表（peer表） bgp转发表 路由表

BGP版本

BGPv4：IPv4

BGPv4+：IPv6

MP-BGP：协议npls

M-BGP：多播

BGP优缺点

优点

1EGP路由表只能存储千条信息，bgp可上万条。

2丰富的路径属性，13条选路原则，提供了更丰富的路由操控策略。

缺点

1对路由器性能要求较高。

2配置程序比较复杂。

3一般小网络不建议使用bgp

AS 和 邻居类型

bgp AS 一台路由器只能属于一个

公网AS1-65411 65412-65535私有AS

iBGP：处于同一AS内的BGP路由器所建立的BGP对等体关系

eBGP：不同AS内的BGP路由器建立的…

BGP消息

open：建立成功，会互相发送open信息(类似hello)协商邻居关系的各项参数，等待对方确认。

keeplive：维持对等体(类似邻居)关系的建立，60s发一次，180s后peer关系结束

update：不会周期性更新，只会增量更新或触发更新。

Notification：当BGP检测到错误状态时，就向对等体发出Notification消息，之后BGP连接会立即中断。

peer关系建立过程

IDLE（空闲状态）：本地路由向外发送tcp(端口179)请求，等回复

connect状态：tcp建立

active：建立不成功，再次请求tcp连接的建立（回到IDLE）。

open sent：建立成功，发送open消息

openconfirm：收到对方的信息进行参数对比，参数一致则建立完成。

established：连接建立。。update。。keepalive。。

bgp的防环机制

iBGP-水平分隔原则

BGP对等体路由器收到iBGP路由，不会发送给其他iBGP对等体路由器。

但是纯水平拓扑上该原则会导致路由不被传递学习。两个解决方案

全互联拓扑解决方案：所有路由器间都建立iBGP关系。（垃圾方案）

路由反射器解决方案：

eBGP

基于AS PATH信息的检测。接收到来自于eBGP peer的路由更新，更新所带AS-PATH 信息包含了当前设备AS号，则该路由更新忽略。

特殊拓扑下路由反射器的防环机制

originate id：起源id。路由器将设备的RID设置为起源ID。起源ID和当前设备RID一样，则路由更新不学习。

cluster id：簇id。管理员在路由反射器上设置，路由更新携带的簇ID一致，则路由更新不学习。

同步与下一跳判断

路由更新来自于iBGP peer时才考虑同步和下一跳。

同步机制

对于当前启用BGP的设备，路由更新对应的网络底层协议也必须能学习到，这才叫同步。同步是为了避免路由黑洞。

现实中同步被关闭，路由黑洞问题无法避免，但利用MPLS能解决。

bgp路由黑洞

路由更新的传递设备也就是数据传输的沿途设备，并没有参与BGP ，学习该路由，所以数据抵达后，只能丢包。

解决方法：

1重分发(不建议，会增大路由表)

2全互联的网络拓扑(两两之间建立ibgp对等体)

下一跳判断

来自于iBGP peer的路由更新，如果下一跳不可达，路由更新不学习。

路由源于AS外，通告者是eBGP peer，下一跳就是eBGP peer地址。

路由源于AS外，通告者是iBGP peer，下一跳依旧是eBGP peer地址。此时下一跳可能不可达。

路由源于iBGP peer，通告者是iBGP peer，下一跳就是iBGP peer地址。

BGP的路由更新属性

  1. 公认必选

    NEXT-HOP：下一跳

    AS-PATH：AS路径信息

    Originate code：起源代码

    ​ 描述了目标网络进入BGP的方式（I E ?）

    ​ I ：目标网络源于底层协议通过network进入BGP

    ​ E：EGP共享的目标网络信息

    ​ ?：外部路由以重分发的方式进入BGP

  2. 公认任选（自选）

    LP：本地优先

    ​ 路由更新在IBGP peer间传递时，LP值才出现，且为100

    ​ 作用：影响iBGP peer 出站路径的选择，LP参数越大越优。

    automatic aggregate：原子聚合

    ​ BGP发生了自动聚合或手动聚合时，聚合后网络携带该属性。

    ​ 原子聚合属性用于提醒更新接受者，路由更新目标网络可能丢失了部分原有属性。

  3. 任选可传递

    Aggregated：被聚合属性——聚合后的网络携带的属性，表明该网络是聚合来的。

    Community：团体属性 是携带特定策略的标记

    ​ 公认团体属性：

    ​ Internet：所有的BGP路由默认携带该属性，该属性对应的策略是，共享给所有的BGP peer。

    ​ no-export：不导出，携带该属性的路由更新不共享给eBGP peer。

    ​ no-advertise：不通告，携带该属性的路由更新不共享给任何BGP peer。

    ​ local-as：本地as，携带该属性的路由更新不共享给联盟内其他eBGP peer。

    ​ 自定义团体属性：

    ​ AA:NN

    ​ AS号：标记值

  4. 任选不可传递（属性保留，但值为空）

    MED：多出口距离

    ​ eBGP间路由共享携带该属性，MED值用于影响eBGP peer入站路径选择。MED值默认为0，越小越优。

BGP选路原则

选路原则决定了BGP按什么标准进行路由更新属性的比较

  1. 优选拥有最高权重值的路径 weight

    用于影响当前路由器出站路径选择

    默认32768，越大越优

    不同类型的对等体出现，需要影响路径抉择，权重更优。

  2. 优选LP大的

  3. 优选起源于本地的路径

    next-hop=0.0.0.0

  4. 优选拥有最短AS-PATH 路径信息的

  5. 比较Origin属性<IGP优先于EGP优先于Incomplete>

  6. 优选拥有最佳MED值的

  7. eBGP peer 的路径优于 iBGP peer的路径

  8. 优选离下一跳更近的路径

  9. 针对eBGP peer的路由，通过命令行允许多条路径，成为最佳进入路由表

  10. 优选最早被学习的路由，只针对 eBGP peer

  11. 优选拥有最小RID的BGP peer提供的路由

  12. 优选拥有最短cluster-id 的路径

  13. 优选拥有最低peer地址的邻居提供的路径

自动汇总 auto-summary

auto-summary 关闭的情况下，BGP 处于精确宣告状态

auto-summary 开启的情况下，BGP 处于非精确宣告状态

针对外部路由，auto-summary 开启，外部路由自动汇总，否则外部路由就明细进入。
