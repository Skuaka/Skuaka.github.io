Linux备忘录

系统安装后的配置

  • 关闭防火墙

    chkconfig iptables off

  • 关闭selinux

    /etc/selinux/config

    SELINUX=disabled

  • chkconfig NetworkManager off

    systemctl disable NetworkManager.service

  • reboot

自动加载文件系统/etc/fstab

  1. 文件系统对应的设备名。这一列内容可以是分区或设备
  2. 挂载点
  3. 文件系统格式。如 ext3， vfat 等
  4. 文件系统挂载参数。不同的文件系统可能有不同的参数
  5. 备份标志。设定为 1 时，表示备份该文件系统；0 表示不备份
  6. 系统启动时，文件系统的检查顺序。只有根分区才可以设定为 1，其余需要检查的分区应该设定为 2，设定为 0 表示不检查文件系统

  1. IP地址配置文件 /etc/sysconfig/network-scripts/ifcfg-网卡
  2. 主机名映射文件/etc/hosts
  3. 主机名文件/etc/sysconfig/network
  4. DNS解析文件/etc/reslov.conf

软件包管理

  5. rpm -ivh 包名

逻辑卷管理

新建逻辑卷

  1. 磁盘分区

    fdisk /dev/sdb # 8e Linux LVM

  2. 创建PV

    pvcreate /dev/sdb1 /dev/sdb2

  3. 创建VG

    vgcreate myVG /dev/sdb1 /dev/sdb2

  4. 创建LV

    lvcreate -L 400M -n vol01 myVG

  5. 创建Filesystem

    mkfs.ext3 /dev/myVG/vol01

删除逻辑卷

  1. 首先将正在使用的逻辑卷卸载掉 umount 命令
    1. 将逻辑卷先删除　　 lvremove 命令
  2. 删除卷组　　 vgremove 命令
  3. 最后再删除物理卷　　 pvremove 命令

扩充逻辑卷

  1. 将要添加到VG的硬盘格式化成PV

    pvcreate /dev/sdd

  2. 将新的PV添加到指定的VG当中

    vgextend xiaoluo /dev/sdd

  3. 扩充逻辑卷

    lvextend -L +1G /dev/xiaoluo/mylv

  4. 更新文件系统

    resize2fs /dev/xiaoluo/mylv

缩容逻辑卷

逻辑卷的缩小操作必须离线执行，要先卸载掉逻辑卷才可以

第一种

  1. 卸载已经挂载的逻辑卷　　通过 umount 命令

  2. 缩小文件系统(会提示需要运行fsck来检查文件系统)

    resize2fs /dev/xiaoluo/mylv 10G

  3. 缩小LV

    lvreduce -L -4G /dev/xiaoluo/mylv

  4. 缩小完逻辑卷以后，再将逻辑卷挂载

第二种

  1. 卸载逻辑卷　　通过 umount 命令

  2. 将一个PV从指定卷组中移除掉

    vgreduce xiaoluo /dev/sdd

  3. 将PV删除掉

    pvremove /dev/sdd

网卡绑定

  1. 关闭NetworkManager服务

  2. 创建bonding.conf文件在/etc/modprobe.d/目录中alias bond0 bonding

  3. 创建ifcfg-bond0文件在/etc/sysconfig/network-scripts/目录中

    DEVICE=bond0

    NAME=bond0

    TYPE=Ethernet

    ONBOOT=yes

    BOOTPROTO=static

    IPADDR=192.168.50.111

    NETMASK=255.255.255.0

    GATEWAY=192.168.50.1

    BONDING_OPTS=”mode=1 miimon=100”

  4. 修改真实网卡的配置 /etc/sysconfig/network-scripts/ifcfg-要加入的两块网卡

    DEVICE=eth0

    NAME=eth0

    ONBOOT=yes

    BOOTPROTO=none

    MASTER=bond0

    SLAVE=yes

  5. 重启网络服务service network restart

  6. 查看状态cat /proc/net/bonding/bond0

NFS服务配置

使用NFS服务，至少需要启动以下3个系统守护进程

  1. rpc.nfsd
  2. rpc.mountd
  3. portmap（rhel5） rpcbind（rhel6）

\d命令

该命令将显示每个匹配关系（表，视图，索引，序列）的信息，包括对象所有的列，它们的类型，表空间（如果不是默认的）和任何特殊属性（如NOT NULL或默认值等）。与唯一约束相关的索引，规则，约束，触发器也同样会显示出来。如果关系是一个视图，还会显示视图的定义。

  1. 如果\d命令什么都不带，将列出当前数据库中的所有表。
  2. \d后面跟一个表名，表示显示这个表的结构定义。
  3. \d也可以显示索引信息
  4. \d后面可以跟一通配符”*“或”?”

  5. \d+命令，该命令将显示比\d命令更详细的信息

  6. 匹配不同对象类型的\d命令

    如果想只显示匹配的表，可以使用\dt命令

    如果想只显示索引，可以使用\di命令

    如果想只显示序号，可以使用\ds命令

    如果想只显示视图，可以使用\dv命令

    如果想只显示函数，可以使用\df命令

  7. 如果想显示SQL已执行的时间，可以用\timing命令

  8. 列出所有的schemas可以使用\dn命令

  9. 显示所有的表空间可以用\db命令

    表空间就是一个目录，放在这个表空间的表，就是把表的数据文件放到这个表空间下。

  10. 列出数据库所有角色或用户\du或\dg

  11. \dp或\z命令用于显示表的权限分配情况

Mac 下 PostgreSQL 配置

在 mac 下，可以利用 homebrew 直接安装 PostgreSQL：

brew install postgresql

初始数据库：

initdb /usr/local/var/postgres -E utf8

上面指定 “/usr/local/var/postgres” 为 PostgreSQL 的配置数据存放目录，并且设置数据库数据编码是 utf8，更多配置信息可以 “initdb –help” 查看。

启动 PostgreSQL：

pg_ctl -D /usr/local/var/postgres -l /usr/local/var/postgres/server.log start

关闭 PostgreSQL：

pg_ctl -D /usr/local/var/postgres stop -s -m fast

创建一个 PostgreSQL 用户

上面的 username 是用户名，回车输入 2 次用户密码后即用户创建完成。更多用户创建信息可以 “createuser –help” 查看。

创建数据库

createdb dbname -O username -E UTF8 -e

上面创建了一个名为 dbname 的数据库，并指定 username 为改数据库的拥有者（owner），数据库的编码（encoding）是 UTF8，参数 “-e” 是指把数据库执行操作的命令显示出来。

更多数据库创建信息可以 “createdb –help” 查看。

连接数据库

psql -U username -d dbname -h 127.0.0.1

远程连接mysql

创建一个和root一样权限的新用户，差别是可以从任何IP登入（@’%’，%匹配任何IP）。

mysql> grant all privileges on *.* to 'skuaka'@'%' identified by '你想设置的密码' with grant option;

  • all privileges：表示将所有权限授予给用户。也可指定具体的权限，如：SELECT、CREATE、DROP等。
  • on：表示这些权限对哪些数据库和表生效，格式：数据库名.表名，这里写*.*表示所有数据库，所有表。如果我要指定将权限应用到test库的user表中，可以这么写：test.user
  • to：将权限授予哪个用户。格式：”用户名”@”登录IP或域名”。%表示没有限制，在任何主机都可以登录。比如：”skuaka”@”192.168.0.%”，表示yangxin这个用户只能在192.168.0IP段登录。
  • identified by：指定用户的登录密码
  • with grant option：表示允许用户将自己的权限授权给其它用户

对用户做了权限变更之后，一定记得重新加载一下权限，将权限信息从内存中写入数据库。

mysql> flush privileges;

可能还要重启一下mysqld服务器，当时直接重启，忘记测试没重启能不能连接了。

service mysqld restart

终于可以远程连接了。

php入门

php四种界定符

  • 标准风格 <?php ?>
  • 短标记风格 <? ?> 默认不支持短标记风格，要在php配置文件中（php.ini）开启支持短标记，然后重启服务器。
  • asp风格 <% %> 默认不支持，要在配置文件中开启。
  • script风格 <script language="php"> <script>

php执行原理

客户端向服务器发送一个请求，如果请求的是一个html页面，服务器直接将html页面发送到客户端给浏览器解析；如果请求的是php页面，则服务器会运行php页面然后生成标准的html，将生成的html发送到客户端。

php语法特点

  1. 以分号结尾
  2. 关键字不区分大小写，但是变量名区分大小写。

变量

  1. 变量以$开头，但$不是变量名的组成部分，仅仅表示变量名从这里开始。
  2. &表示地址。&a=&$b //将$b的地址赋给&a
  3. 通过unset()来销毁变量名，值是由php的辣鸡回收机制来销毁。

常量

常量用define()声明。define('name','value');可以通过添加第三个参数为true来表明不区分大小写，false表明区分大小写（默认情况）。

数据类型

字符串

单引号字符串不修改内容，双引号字符串会将变量替换为值，如果变量名后面有其他内容，要用大括号将变量括起来："{$name}_other_string"

数组

  • 索引数组，通过元素的位置做下标。默认从0开始，每次增长1，可以更改起始下标。

  • 关联数组，通过字符串做下标
