CSS入门

CSS简介

CSS(cascading style sheet) 层叠样式表

语法：选择器{属性名:属性值; 属性名:属性值; }

说明：

  • 一个css样式包括选择器和格式声明语句
  • 选择器就是选择给哪个html标签加样式
  • 格式声明语句要用{ }，包括属性名：属性值；
  • 属性名，w3c 已经定义好，直接使用
  • 属性值不用双引号
  • 属性值多以像素px为单位，通常情况必须加单位（html属性不加单位）

CSS样式表

如何在html结构中引入css样式

  1. 行内样式表：

    每一个标签都有style属性

    ​ 格式：<标签 style=“属性名:属性值; 属性名:属性值;” >内容</标签>

    只能一个一个设置，麻烦，临时测试用

  2. 内嵌样式表：

    格式：在head标签中输入 (在xhtml中必须写type属性)

  3. 外部样式表：

    格式：<link rel="stylesheet"href="外部样式表文件 xx.css"/>

  4. 导入样式表：

    管理css样式，大型网站需要

    格式: @import url(另外一个css样式)

    注意: @import 是css样式标签，必须放到css文件中样式表的最上端

CSS选择器

基本选择器

  • 标签选择器 body{} p{} div{}
  • 类选择器

    每一个标签都有一个class属性<label class="myclass"></label>

    类用 .来表示 例如.myclass{ } .page_header{ }

  • id选择器

    每一个标签都有一个id属性<label id="myid"></label>

    id用#来表示 例如#myid{ }

    注意：id只能唯一，通常id给JavaScript用，不是用来设置样式的，如果想设置样式，用类选择器

  • 通用选择器

    给所有的html标签加样式

    语法：*{ }

    ie6版本不支持

复合选择器

  • 多元素选择器

    多个标签共有的属性和属性值，放到一起

    语法：选择器1,选择器2,选择器3……{ }

  • 后代元素选择器

    给html的后代元素（不仅是直接子元素）加样式

    语法：选择器1 选择器2 选择器3{ }

    以空格隔开，表明选择器1里面的选择器2里面的选择器3

  • 子元素选择器

    给html的直接子元素加样式

    语法：选择器>选择器{ }表明某个元素里面的第一层元素

固定定位 fixed

  • 相对于浏览器窗口来进行的定位

  • 层级要比普通标签高

相对定位 relative

  • 相对定位占空间（自身原来的空间）

  • 绝对定位不占空间

  • 层级高于普通元素

  • 不结合定位坐标，就是在原来的位置

  • 结合定位坐标，就是以祖先元素（设置绝对定位，相对定位）作为坐标的参考点

    如果祖先没有设置定位，将会一直往上寻找，直到找到body，就以body来进行定位，相对于整个窗口

CCIE NAT 地址转换协议

NAT 地址转换协议

优缺点

优点
在地址重叠时提供解决方案

在网络发生变化时避免重新编织

最大作用为可将私网地址转换为公网地址，提升了ip地址空间的使用率，这也是ipv4至今还未耗尽的原因

缺点

增加交换延迟

无法进行端到端ip跟踪

有些应用程序无法正常运行

网络地址转换类型

静态NAT：在本地地址和全局地址直接进行一对一的映射。必须为网络中的每台主机提供一个公有ip地址。

动态NAT：将未注册的ip地址映射到注册ip地址池中的一个地址。必须有足够多的公有ip地址，让连接到因特网的主机都能够同时发送和接收分组。

NAT重载：最常用的NAT类型

也被称为端口地址转换(PAT)

利用源端口将多个非注册ip地址映射到一个注册ip地址(多对一)

使用传输层端口号来标识主机，因此理论上最多可让大约65 000台主机共用一个公有ip地址

本地地址：进行NAT转换前的地址

内部本地地址：转换前的内部源地址，主机的私网地址

外部本地地址：转换后的目标地址，与ISP相连的路由器接口的地址，通常为共有地址

全局地址：进行NAT转换后的地址，通常为因特网共有地址

内部全局地址：转换后的源地址

外部全局地址：转换前的外部目标地址

配置方式

接口角色：
inside：连接在内部广播域
outside：连接公网

转换接口角色
int xx
ip nat inside/outside

静态NAT

ip nat inside source static 内部ip 外部ip

动态NAT

ip nat pool todd 170.168.2.3 170.168.2.254 netmask 255.255.255.0

创建一个地址池todd，用于将全局地址分配给主机

ip nat inside source list 1 pool todd

将与access-list 1匹配的ip地址转换为ip地址池todd中的一个可用地址

PAT(NAT重载)

与动态NAT相比，PAT地址池只包含一个ip地址，且配置命令末尾包含关键字overload

ip nat pool globalnet 170.168.2.1170.168.2.1 netmask 255.255.255.0

ip nat inside source list 1 pool overload

(重复使用，如果地址池有其他地址，也可以使用它，适合大型网络，必须重载多个公有ip地址)

验证与修改

show ip nat translation

debug ip nat

show ip nat statistics

命令claer ip nat translations 清除转换条目，可以使用全局地址、本地地址、tcp(udp)端口、*（星号，清除所有转换条目）

只要NAT表中有包含地址池中地址的转换条目，思科ios就不允许修改或删除相应的地址池

转换条目超时时间：默认为24小时，可以使用命令ip nat translation timeout修改。如果定时器到期，相应的转换条目将从NAT表中删除，动态分配给它的地址将归还给地址池。

DFS BFS

DFS思路

BFS和DFS选择

1.BFS是用来搜索最短径路的解是比较合适的，比如求最少步数的解，最少交换次数的解，因为BFS搜索过程中遇到的解一定是离根最近的，所以遇到一个解，一定就是最优解，此时搜索算法可以终止。这个时候不适宜使用DFS，因为DFS搜索到的解不一定是离根最近的，只有全局搜索完毕，才能从所有解中找出离根的最近的解。（当然这个DFS的不足，可以使用迭代加深搜索ID-DFS去弥补）
2.空间优劣上，DFS是有优势的，DFS不需要保存搜索过程中的状态，而BFS在搜索过程中需要保存搜索过的状态，而且一般情况需要一个队列来记录。
3.DFS适合搜索全部的解，因为要搜索全部的解，那么BFS搜索过程中，遇到离根最近的解，并没有什么用，也必须遍历完整棵搜索树，DFS搜索也会搜索全部，但是相比DFS不用记录过多信息，所以搜素全部解的问题，DFS显然更加合适。

access-list命令

（1）标准访问列表

access-list access-list-number { permit | deny } {source [source-wildcard] | any}

命令解释如下。

access-list：访问列表命令。

access-list-number：访问列表号码，值为1~99.

permit：允许。

deny：拒绝。

source：源IP地址。

source-wildcard：源IP地址的通配符。

（2）扩展访问列表

access-list access-list-number { permit | deny } { protocol \ protocol-keyword } { source [ source-wildcard ] | any } { destination destination-wildcard } | any }[protocol-specific options][log]

命令解释如下。

access-list-number：访问列表号码，值为100~199.

protocol \ protocol-keyword：可使用的协议，包括IP、ICMP、IGRP、EIGRP、OSPF等。

destination destination-wild：目的IP地址，格式与源IP地址相同。

protocol-specific options：协议制定的选项。

log：记录有关数据报进入访问列表的信息。

vtp（虚拟传输协议）

用于同步同一网络中交换机上网络的信息（减小庞大的配置量）

生效的前提：trunk正常工作

vtp mode sever/client/transparent

服务器：NVRAM（可寄存性存储器）创建、修改、维护、传递、删除、同步vlan信息

客户端：RAM(重启后要重新同步） 传递、同步vlan信息

透明模式：NARAM 和服务器一样除了不同步，并且本地创建、修改、维护、删除的vlan 信息只在本地有效。

sh int tr确保生效

sh vtp status

其中Configuration Revision（配置修订号）（类似书的修订号，修改一次增加一次，不能减小）越大越好，同步大的。加入新的交换机时可用透明模式先测试。

CCIE STP生成树协议

STP生成树协议 7.27

一个广播域中选举一个生成树。

广播风暴：绝不停止。

重复帧。

mac地址表的不稳定。

1一个网络中选取一个根桥

2根桥上所有端口都为指定端口

3非根桥上选取一个根端口

4每一段选举一个指定端口

5非指定端口为阻塞端口

端口状态

Blocking（阻塞状态）：此时，二层端口为非指定端口，也不会参与数据帧的转发。该端口通过接收BPDU来判断根交换机的位置和根ID，以及在STP拓扑收敛结束之后，各交换机端口应该处于什么状态，在默认情况下，端口会在这种状态下停留20秒钟时间。

Listening（侦听状态）：生成树此时已经根据交换机所接收到的BPDU而判断出了这个端口应该参与数据帧的转发。于是交换机端口就将不再满足于接收BPDU，而同时也开始发送自己的BPDU，并以此通告邻接的交换机该端口会在活动拓扑中参与转发数据帧的工作。在默认情况下，该端口会在这种状态下停留15秒钟的时间。

Learning(学习状态)：这个二层端口准备参与数据帧的转发，并开始填写MAC表。在默认情况下，端口会在这种状态下停留15秒钟时间。

Forwarding（转发状态）：这个二层端口已经成为了活动拓扑的一个组成部分，它会转发数据帧，并同时收发BPDU。

Disabled（禁用状态）：这个二层端口不会参与生成树，也不会转发。

基本规则

  • 在一个脚本程序中执行另外一个脚本程序要比执行一个函数慢的多

  • 函数的定义方式

  • shell中不存在前向声明，要把所有函数定义都放在任何一个函数调用之前。

  • 当函数被调用时，脚本程序的位置参数($*,$@,$#,$1,$2等)会被替换为函数的参数，这就是读取传递给函数的参数的方法。当函数执行完毕后，这些参数会恢复为之前的值。

  • return让函数返回数字值

  • 使用local关键字在shell函数中声明局部变量，如果一个局部变量名字和全局变量名字相同，在函数的作用范围内，前者就会覆盖后者。

shell常见命令

shell脚本内部可以执行两种命令：

  • 外部命令(external command): 可以在命令提示符中执行的普通命令
  • 内部命令(internal command): 内部命令是在shell内部实现的，不能作为外部程序被调用。（然而，大部分的内部命令同时也提供了独立运行的程序版本——属于POSIX规范）

常用命令：

  1. break 跳出循环

  2. 冒号(:)命令是一个空命令。

    它偶尔用于简化条件逻辑，相当于true的别名，while ：实现了无限循环。由于它是内置命令，所以运行的比true快，但是输出可读性差。

  3. continue 跳到下一次循环

  4. 点(.)命令，用于在当前shell中执行命令。

    ​ 通常，当一个脚本执行一条外部命令或脚本程序时，它会创建一个新的环境（一个子shell），命令将在这个行环境中执行，在命令执行完毕后，这个环境被丢弃，留下退出码返回给父shell。

    ​ 所以，当你要用一个脚本来为后续执行的命令设置环境时，这命令就非常有用，在这种意义上，点命令的作用类似于C/C++的#include指令。

    ​ 示例：假设有两个包含环境设置的文件，他们分别针对两个不同的开发环境。

    （1）为了设置老的环境，可以使用文件classic_set

    （2）对于新命令，使用文件lastest_set

    （3）可以通过将这些脚本程序和点命令结合来设置环境

  5. echo命令

    ​ 虽然，X/Open建议在现代shell中使用printf命令，但是使用echo方便。

    ​ 如何去掉换行符？

    • echo -n "str" -n参数去掉换行符

    • echo -e "str\c" -e参数确保启用了反斜线转义字符的解释(\c代表去掉换行符)

      老版本bash默认开启反斜线转义字符的解释，但最新版本的bash不一定开启。

    如果需要脚本兼容UNIX系统并且需要删除换行符，最好坚持使用printf命令。

    如果只运行在Linux和bash上，那么echo-n是不错的选择。
