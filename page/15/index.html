CCIE STP生成树协议

STP生成树协议 7.27

一个广播域中选举一个生成树。

广播风暴：绝不停止。

重复帧。

mac地址表的不稳定。

1一个网络中选取一个根桥

2根桥上所有端口都为指定端口

3非根桥上选取一个根端口

4每一段选举一个指定端口

5非指定端口为阻塞端口

*粗体为生成树定义的交换机接口角色

根端口：距离根桥距离最短的端口。开销值一样的话就选最小的发送者id。只在非桥根上选举，并且只选一个。

指定端口：可以执行数据的转发。一台交换机上可以有多个指定端口。根桥上所有端口都是指定端口。

阻塞端口：逻辑上的阻塞，不执行数据的转发，但会持续侦听bpdu信息。

BPDU(桥协议数据单元)只允许根桥产生

root id

优先级：(默认32768，越小越优，更改 优先值时必须为4096的倍数)。

背板mac：标识一台交换机，背板mac小的为优。

bridge id

优先级+背板mac：和根id一样

cost

开销值

port id

优先级(默认128，小优，更改时必须为64的倍数)

端口mac：发送者的端口mac

PVST：每vlan生成树，在优先级插入vlan的信息。优先级+vlan编号。

端口状态

Blocking（阻塞状态）：此时，二层端口为非指定端口，也不会参与数据帧的转发。该端口通过接收BPDU来判断根交换机的位置和根ID，以及在STP拓扑收敛结束之后，各交换机端口应该处于什么状态，在默认情况下，端口会在这种状态下停留20秒钟时间。

Listening（侦听状态）：生成树此时已经根据交换机所接收到的BPDU而判断出了这个端口应该参与数据帧的转发。于是交换机端口就将不再满足于接收BPDU，而同时也开始发送自己的BPDU，并以此通告邻接的交换机该端口会在活动拓扑中参与转发数据帧的工作。在默认情况下，该端口会在这种状态下停留15秒钟的时间。

Learning(学习状态)：这个二层端口准备参与数据帧的转发，并开始填写MAC表。在默认情况下，端口会在这种状态下停留15秒钟时间。

Forwarding（转发状态）：这个二层端口已经成为了活动拓扑的一个组成部分，它会转发数据帧，并同时收发BPDU。

Disabled（禁用状态）：这个二层端口不会参与生成树，也不会转发。

软件防火墙：省查木马或病毒：保护的是系统

网络的边缘部署防火墙，一般放在 PPPOE 拨号设备的身后，关键节点之间。

防火墙的基础知识

防火墙工作模式

路由模式：（默认模式）防火墙读取 IP 包头，进行网络层的保护，支持动态路由协议和静态路由。

交换模式：

防火墙的多模式和单模式

多模式支持虚拟防火墙，即把一台防火墙配置成多台逻辑防火墙。

单模式是默认模式。

防火墙接口的类型

inside：内部接口

通常连接的是信任网络即内网

outside：外部接口

通常连接的是非信任公共网络，比如 ISP 或 其他组织网络

DMZ：非军事区域

通常数据中心放在 DMZ 区域

防火墙接口等级和访问规则

inside 接口安全等级：100

outside 接口安全等级：0

DMZ 接口安全等级：50、0

高等级接口接收流量穿越防火墙从低等级接口发送出去，必须经过 NAT。

低等级接口接收流量穿越防火墙从高等级接口发送出去，默认不允许。

除非是返回流量或被管理员手动放行。

相同安全级别的接口相互通讯，默认不允许通讯，除非管理员放行。

BGP 的路由更新过滤

route-map、prefix-list、filter-list（筛选列表）

利用 route-map

可以是 in 方向，也可以是 out 方向。

route-map 有隐藏的 deny all。

利用 prefix-list

route-map 不光可以完成过滤，还可以对放行的网络设置策略。

prefix-list 只能完成过滤。

prefix-list 也有默认的 deny all。

prefix-list 的放行所有 ip prefix-list x permit 0.0.0.0/0

利用 正则表达式

正则表达式匹配击中的是 AS 路径信息。

放行所有：ip as-path <id> permit _

BGP 的公认团体属性限制路由更新的发送

路由更新携带 no-advertise，路由更新被接收后不发送给任何 BGP peer。

路由更新携带 no-export 或 local-as，路由更新被接收后不发送给任何 eBGP peer。

查看BGP转发表中携带特定团体属性的路由更新show ip bgp community <no-export/no-advertise>

CCIE IS-IS

IS-IS（中间系统到中间系统）

数据传递方式

  • 面向连接 tcp/ip

  • 非面向连接 clns/clnp 非连接的网络服务(负责控制数据传递过程)/非连接的网络协议(网络地址标识)

    最后，ip获得广泛使用，但是CLNS的理念也得到认可，因此保留下来，叫做UDP。

IS-IS术语

  • IS：中间系统 即 路由器router

  • ES：终端系统 （但是现在已经没有使用clnp标识的设备了）

  • clnp：标识

    NET：网络实体标识（网络+主机）128bit

    clnp：clnp地址 NSAP网络服务应用标识

clnp地址解释

49.0001.0000.0000.0001.00

49.0002.0000.0000.0002.00

49.0101.0000.0000.0001.00

49：AFI地址格式标识 A地址 F格式 I授权标识 数字由官方的表定好。47，48，49用于私网。

0101 01/01 域/区域 （区域包含在域内）这里为两个十六进制数

0000.0000.0001 mac地址

00 服务号（作用类似于端口号）

IS-IS 域、区域{骨干区域，常规区域}

域：AS

区域：0，1，2 ……

路由器角色类型

  • OSPF{角色：DR BDR DR-other，类型：ABR ASBR}
  • IS-IS {角色：Dis(不完整)，类型：L1 L2 L1/L2}

伪节点

  • OSPF：DR{减少邻接关系，提高更新效率}
  • IS-IS：Dis

IS-IS类型

  • L2 骨干区域
  • L1 常规区域
  • L1/L2(默认类型) 连接两个区域

OSPF LSA 特殊区域

LSA：Link-State Advertisement 链路状态广播

DBD：数据库描述报文

LSR：链路状态查询

LSU：链路状态更新

OSPF七种LSA类型

  • LSA1（Router LSA）

    起源：区域内每台设备都能产生LSA1

    内容：节点信息{RID network cost}

    扩散：不允许穿越ABR，也就是区域内扩散。

    路由：区域内路由

  • LSA2（Network LSA）

    起源：只有DR可以产生

    内容：拓扑信息{节点的直连情况，谁是DR}

    扩散：区域内

  • LSA3（Network summary LSA）

    起源：ABR

    内容：区域网络信息

    扩散：整个OSPF AS

    路由：区域间路由

  • LSA4（ASBR summary LSA）

    起源：ASBR所在域的ABR

    内容：ASBR的RID

    扩散：整个OSPF

    路由：外部{OE1，OE2}

  • LSA5（Autonomous system external LSA）

    起源：ASBR

    内容：外部路由

    扩散：整个OSPF

    路由：外部{OE1，OE2}

OSPF LSA 特区

LSA6 MOSPF专用（废除）

LSA10 口袋路由

LSA11 标记

ISIS的汇总及重分发

汇总和重分发最好做在拥有L2角色的设备上。

汇总后的路由和重分发的路由都是L2设备。

L1设备上完成重分发和汇总的解决方法

  • L1设备上重分发

    重分发时指定外部路由类型改为L1

  • L1设备上做汇总

    被汇总的网络必须以外部路由形式进入ISIS，再汇总才生效

    ！！注意

    ISIS的外部路由类型是L2，那么接受者不累加接口开销

    ISIS的外部路由类型是L1，那么接受者会累加接口开销

ISIS的路由泄露

ISIS路由泄露 指 将指定的L2数据库目标网络信息转化成L1的，发送给L1设备。

路由泄露的配置流程

步骤1：L1/L2 设备上编写 ACL，匹配被泄露的目标路由

步骤2：L1/L2 设备上编写 ROUTE-MAP 引用 ACL。

步骤3：L1/L2 设备上做 ISIS 到 ISIS 的重分发，并且指定是利用 route-map 完成 L2 数据库条目到 L1 数据库条目的泄露。

2：IS-IS的发展历程

同时进行的，没有谁早谁晚的说法。

OSI七层模型 传输层 面向链接TCP 无链接UDP

OSI中后期产物 计算机网络最早的发展阶段提出的是CLNS和CONS

CLNS和CONS算是OSI的前身

N代表网络，S代表服务，C代表链接 O代表Of“有链接的网络服务”类似于TCP L代表Less “无链接的网络服务”

基于对CONS的研究，延伸出了IP【网络标识】和TCP

TCP数据传输控制 IP网络标识

因此开发出了动态的路由协议：RIP EIGRP OSPF BGP

另一阵营：CLNS坚持使用无链接的网络协议

开发出了CLNP：CLNS网络标识

还需要的机制：CLNS【名字重复】IS-IS

OSPF为IP服务，IS-IS为CLNP网络服务。

这些都是不同组织在同时发展的，最后CONS阵营的方案得到公司的认可。

最后，ISO组织提出了OSI七层模型，但是CLNS的理念也得到认可，因此保留下来，叫做UDP

最后，IS-IS就感觉要消失了

这是为了统一行业标准。

OSPF虚链路：①克服孤岛区域：0，1，2区域不互相学路由

②两边区域0，中间区域1，但是两边区域0不互相学路由。

IS-IS没有这个问题。

外企推运营商用IS-IS 思科排第一

割接：干掉原有协议，换新协议

两种方法：①升级OSPF，使其拥有IS-IS特性

②直接用IS-IS取代OSPF

两种方法都不太科学……

所以我们现在使用的IS-IS，并不是纯IS-IS

VPN

虚拟的私有网络，通过组建虚拟的专网实现数据的安全传递。

VPN的分类

OSI 参考模型分类

  • 应用层VPN：开源的第三方VPN软件（open vpn）
  • 传输层VPN：SSL VPN（安全套字节层 VPN）主要用于HTTP的安全加固和防护。
  • 网络层VPN：IPSEC VPN（ip安全VPN）、DM-VPN（动态多点VPN，思科私有）
  • 数据链路层VPN：L2TP（层2隧道VPN）、PPTP（点到点隧道VPN）微软思科联合开发
  • 物理层VPN：专线

实际应用分类

  1. site to site：站点到站点：L2L（lan to lan）

    IPSEC VPN、DM-VPN、MPLS VPN

  2. remote vpn：远程接入VPN

    L2TP、PPTP、SSL VPN、EZ-VPN（思科私有）、anyconnect VPN（任连VPN）

MPLS VPN

MPLS VPN是在BGP peer之间建立（peer to peer VPN），利用了BGP的对等体共享了企业内网路由信息，构建专有通路，实现安全的通讯。

MPLS VPN术语

  • P-NETWORK：运营商核心网络

  • PE 设备：运营商边界设备

  • P 设备：运营商核心设备

  • C-NETWORK：客户网络

  • CE 设备：客户网络边界路由器

  • VRF：虚拟路由转发表

    路由器拥有一个全局路由表和若干个VRF，每个VRF对应到一个C-NETWORK。

    作用：存储不同 C-NETWORK 的路由，保障这些路由不影响全局路由表，而且不会冲突，且相互独立。

    • VRF 的 RD 值：路由区分值

      RD 值书写格式：XX:YY

      不同 VRF 的路由进入BGP 后，利用 RD值进行区别。

      MPLS VPN peer 之间建立的是 MP-BGP peer 关系，MP-BGP 提供专门的转发表，存储来自不同 VRF的路由。

    • VRF 的 RT 值：路由目标值

      RT 值书写格式：XX:YY

      RT 值分为导入和导出两个方向：

      导入值是用来确认是否学习远端的路由

      导出值是路由被共享时携带

      导入和导出值可以是多个

  • MP-BGP VPNv4 更新：即 MP-BGP 的路由更新

MPLS VPN 的双标签

栈底标签：PE 设备上 MP-BGP 给予的，用来帮助数据寻找正确的 VRF路由

栈顶标签：MPLS LDP 给予的，用来穿越P-NETWORK。

安全加密

GET-VPN（隧道加密传输VPN，思科私有）专门在MPLS VPN 基础上加密数据。

跨域MPLS VPN 解决方案详解

方法一：option A：背靠背：back to back

  1. 步骤1：ASBR 上创建 VRF，设定 RD 和 RT，将直连接口划入 VRF。

    R3、R4 创建 VRF 的原因：

  • 利用 VRF 接收来自路由反射器的路由更新，当前 ASBR 设备学习本 AS 站点路由。

  • 创建基于 VRF 的 EBGP PEER 关系，帮助当前 ASBR 设备学习对端 C-NETWORK 路由，并且装载进 VRF 表。

    注意！！ PE 上启用 VRF，切记要将 VRF 和 BGP 相互做重分发，如果 PE 和 CE 间是 BGP，那么来自 CE 的路由更新直接进 PE 的 MP-BGP 转发表。

  • ASBR 创建 VRF 给对端 C-NETWORK 路由新的栈底标签。

方法一如何传递数据

方法一的特点：ASBR 间只有传统的 EBGP peer，ASBR 间没有 MPLS，ASBR 间数据最后是 IP 形式穿越。

方法二：option B 2a

步骤一：ASBR上关闭 RT 过滤功能，在不创建 VRF 的情况下学习 MP-BGP 路由更新，达到学习各自 AS 内部路由的目的

此时栈底标签也跟着共享过来了

步骤二：ASBR 间利用直连网络建立 MP-BGP 关系

方法三：option B 2b

步骤1：ASBR 自身关闭 RT 筛选

步骤2：ASBR 利用静态路由指明抵达对端 ASBR 的 loopback 0 口地址

步骤3：ASBR 间启用 MPLS

如上步骤是考虑到 ASBR 间可能非直连，存在路由黑洞，所以利用 MPLS 的标签，关联帮助数据穿越黑洞。

步骤4：ASBR 间利用环回口建立 MP-BGP EBGP peer

步骤5：将 ASBR 上的静态路由重分发进 BGP

方法四：option C

步骤1：ASBR 间建立 BGPv4 EBGP peer，并且给对方共享标签信息，对对方设置权重值为1.

步骤2：ASBR 上将 PE 和 RR 的 lo0 宣告进 BGP

如上步骤是为了 ASBR 交换各自 AS 内路由反射器及 PE 的路由信息，

ASBR 一定要对 AS 内设备配置 NEXT-HOP-SELF。

步骤3：两个不同的 AS 利用各自路由反射器 lo0 建立 MP-BGP EBGP peer 关系，针对对端做 next-hop-unchange。

步骤4：ASBR 上把来自于对端 AS 的路由重分发进底层 ISIS，将原本被 BGP 学习的路由带入 ISIS，使得当前 AS 的 MPLS 可以分配栈顶标签。

方法四特点：不同 AS 路由反射器直接建立 MP-BGP 对等体关系，使得不同 AS 的 PE 似乎和对端是直接交换路由更新。

NAT 地址转换协议

优缺点

优点
在地址重叠时提供解决方案

在网络发生变化时避免重新编织

最大作用为可将私网地址转换为公网地址，提升了ip地址空间的使用率，这也是ipv4至今还未耗尽的原因

缺点

增加交换延迟

无法进行端到端ip跟踪

有些应用程序无法正常运行

网络地址转换类型

静态NAT：在本地地址和全局地址直接进行一对一的映射。必须为网络中的每台主机提供一个公有ip地址。

动态NAT：将未注册的ip地址映射到注册ip地址池中的一个地址。必须有足够多的公有ip地址，让连接到因特网的主机都能够同时发送和接收分组。

NAT重载：最常用的NAT类型

也被称为端口地址转换(PAT)

利用源端口将多个非注册ip地址映射到一个注册ip地址(多对一)

使用传输层端口号来标识主机，因此理论上最多可让大约65 000台主机共用一个公有ip地址

本地地址：进行NAT转换前的地址

内部本地地址：转换前的内部源地址，主机的私网地址

外部本地地址：转换后的目标地址，与ISP相连的路由器接口的地址，通常为共有地址

全局地址：进行NAT转换后的地址，通常为因特网共有地址

内部全局地址：转换后的源地址

外部全局地址：转换前的外部目标地址

配置方式

接口角色：
inside：连接在内部广播域
outside：连接公网

转换接口角色
int xx
ip nat inside/outside

静态NAT

ip nat inside source static 内部ip 外部ip

动态NAT

ip nat pool todd 170.168.2.3 170.168.2.254 netmask 255.255.255.0

创建一个地址池todd，用于将全局地址分配给主机

ip nat inside source list 1 pool todd

将与access-list 1匹配的ip地址转换为ip地址池todd中的一个可用地址

PAT(NAT重载)

与动态NAT相比，PAT地址池只包含一个ip地址，且配置命令末尾包含关键字overload

ip nat pool globalnet 170.168.2.1170.168.2.1 netmask 255.255.255.0

ip nat inside source list 1 pool overload

(重复使用，如果地址池有其他地址，也可以使用它，适合大型网络，必须重载多个公有ip地址)

验证与修改

show ip nat translation

debug ip nat

show ip nat statistics

命令claer ip nat translations 清除转换条目，可以使用全局地址、本地地址、tcp(udp)端口、*（星号，清除所有转换条目）

只要NAT表中有包含地址池中地址的转换条目，思科ios就不允许修改或删除相应的地址池

转换条目超时时间：默认为24小时，可以使用命令ip nat translation timeout修改。如果定时器到期，相应的转换条目将从NAT表中删除，动态分配给它的地址将归还给地址池。

MPLS详解

multiple protocol label switch（多协议标签交换）

MPLS作用

MPLS是一种数据交换机制，基于标签库和数据携带的标签。

进行数据转发，同时支持多种协议。

MPLS在实际工作中，能实现如下两个基本功能：

  1. 数据包的高速转发
  2. 减少参与BGP的设备数量、减少对等体关系、减少不必要的路由学习

MPLS术语

label 标签

总长 32bit 由如下几个部分组成：

前20bit 是标签取值范围

第21bit 是栈底位，栈底位为1则表明该标签已经是最后一个

第22~24bit 是体验位，可以将标签对应的数据分成8个级别，给予不同的传输策略。

最后8bit 是TTL，限制携带标签的数据传递范围。

MPLS 模式

帧模式：frame mode（主流模式），标签被插在帧头的后面，包头前面。

信令模式：cell mode 标签被插在信令头后面。

LSR：标签交换路由器

启用了MPLS的路由器统称LSR。

入站 LSR （inbound LSR）：接收到 IP 数据包，压入标签，传递给 MPLS 网络。

链路中 LSR （link LSR）：接收到携带标签的数据，基于标签进行数据转发。

出站 LSR （outbound LSR）：负责将标签移除，并且变回 IP 数据报文。

压入、交换、移除、弹出

压入：入站LSR将标签压入数据。

交换：链路中LSR基于标签库，进行标签的置换、数据的传递。

移除：出站LSR将数据携带的所有标签全部删除。

弹出：出站LSR将数据携带的栈顶标签弹出，其他标签保留。

LSP：链路标签路径

是MPLS设备传递数据所需的路径。（即路由）

LSP是通过标签的关联产生的。

标签分配、分发、关联、维护协议

TDP：标记分发协议、思科私有

LDP：标签分发协议、公有公开

LDP详解

LDP作用

  1. 利用 hello 消息建立 LDP 邻接关系。
  2. 基于CEF给本地网络分配本地标签。
  3. 将网络对应的标签分发共享给邻居
  4. 基于底层协议完成本地标签与远程共享标签的关联，最终形成LSP。
  5. 负责监控LSP的有效性，并且维护它。

LDP邻接关系的建立

LDP 邻居的发现，通过hello消息，基于UDP，目标端口646。

LDP 利用TCP，目标端口646完成邻接关系建立。

LDP可以设定RID，RID扮演更新源，更新源会成为对端TCP的目标接收地址，所以更新源必须可达。

LDP标签的分配

除BGP路由，LDP可以给本地直连网络以及非直连网络分配标签。

LDP标签的共享

两种形式：下游主动、上游主动。

实际上标签的分享是给所有的邻居，不遵循所谓的上下游。

LDP标签的关联

LDP永远将路由路径下一跳共享的标签与对应网络的本地标签关联。

boundaddress：绑定地址，参与MPLS设备的IP会成为绑定地址和，会跟着本地标签一起共享给LDP邻居，邻居根据底层协议下一跳地址和绑定地址的判断来确认哪个LDP邻居是下一跳，再绑定。

LDP的特殊标签

  • UNTAGE：未标记

出现原因：底层是ospf,本地网络无法找到对应的远程标签，LDP给网络分配untage。

处理方式：目标网络远程标签如果是untage，那么LSR会将所有标签移除，向外传递。

  • POP：倒数第二跳标签

出现原因：当前路由器给本地直连网络分配的本地标签就是POP。

处理方式：如果出站标签是POP，那么LSR 会将栈顶标签弹出。

意义：提高数据转发效率，减少一次标签库查找。

LDP的两张表

LIB：标签信息库

目标网络本地标签和学习到的远程标签存储在这里。（未关联）

LFIB：标签转发信息库

标签关联后的结果 即 LSP 存储在这里。

MPLS的配置流程

步骤1：开启 CEF（默认开启，一般情况不需要此命令）

ip cef

步骤2：设置标签分配、分发关联协议（默认是LDP）

mpls label protocol <ldp/tdp>

步骤3：设置标签取值范围（通常不设置）

mpls label range <xx-yy>

步骤4：设置 LDP 的 RID 即设置更新源（不设置则和OSPF一样自动选取RID）

mpls ldp router-id loopback x [force] //RID 必须可达、force 强制改变 RID

步骤5：接口下启用MPLS

mpls ip

交换机基础

泛洪：除接收端口外，所有端口都向外去发。

mac地址表=cam表

广播域：所能传播的最大范围

冲突域：发生冲突之后所影响的范围

CSMA/CD：载波帧听多链路访问

交换机指示灯：

绿色常亮：接口两端没有问题，但是当前没有数据从中通过

绿色不停地闪：正在传输数据

橘红色慢闪：端口没问题,对端线路或者对端的端口有问题

橘红色快闪：端口和对端连接没有问题，正在协商参数建立

黄色闪动：物理接口有问题，尝试重启。

不闪： 物理接口有问题，无法通信

vlan（虚拟局域网）

作用：分隔广播域

所有接口默认属于vlan1（默认vlan，不可删除）。

vlan ID范围和解释

\1. 0和4095 仅仅限于系统使用 用户不能查看和使用

\2. 1 正常 cisco默认的vlan,用户可以使用但是不能删除

\3. 2–1001 正常 用于以太网的VLAN，用户可以创建。使用。删除。

\4. 1002–1005 正常 用于FDDI和令牌环的CISCO默认VLAN，用户不能删除

\5. 1006–1024 保留 仅仅限于系统使用。用户不能查看和使用。

\6. 1025–4094 扩展 仅仅用于以太网VLAN

vtp（虚拟传输协议）

用于同步同一网络中交换机上网络的信息（减小庞大的配置量）

生效的前提：trunk正常工作

vtp mode sever/client/transparent

服务器：NVRAM（可寄存性存储器）创建、修改、维护、传递、删除、同步vlan信息

客户端：RAM(重启后要重新同步） 传递、同步vlan信息

透明模式：NARAM 和服务器一样除了不同步，并且本地创建、修改、维护、删除的vlan 信息只在本地有效。

sh int tr确保生效

vtp domain 域名

vtp mode xxx（默认为sever）

sh vtp status

其中Configuration Revision（配置修订号）（类似书的修订号，修改一次增加一次，不能减小）越大越好，同步大的。加入新的交换机时可用透明模式先测试。
