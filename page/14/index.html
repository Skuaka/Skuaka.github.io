CCIE IS-IS协议

1：IS-IS中间系统到中间系统

IGPS阵营中的协议

两大阵营：EGPS和IGPS

IGPS 内部网关协议集

IGPS：RIP OSPF EIGRP IS-IS 都是AS内的协议

EGPS：BGP AS间、AS内

IS-IS正在取代OSPF

学习RIP, EIGRP是为了学习OSPF和BGP？

Link State 链路状态路由协议

根据协议自身特点分类：【分类记住路由协议】

距离矢量 RIP 高级距离矢量 EIGRP 不是靠跳数 公式【基于带宽延迟】+Metric值 但是受距离限制 OSPF 链路状态

*为什么要使用IS-IS取代OSPF？

2：IS-IS的发展历程

同时进行的，没有谁早谁晚的说法。

OSI七层模型 传输层 面向链接TCP 无链接UDP

OSI中后期产物 计算机网络最早的发展阶段提出的是CLNS和CONS

CLNS和CONS算是OSI的前身

N代表网络，S代表服务，C代表链接 O代表Of“有链接的网络服务”类似于TCP L代表Less “无链接的网络服务”

基于对CONS的研究，延伸出了IP【网络标识】和TCP

TCP数据传输控制 IP网络标识

因此开发出了动态的路由协议：RIP EIGRP OSPF BGP

另一阵营：CLNS坚持使用无链接的网络协议

开发出了CLNP：CLNS网络标识

还需要的机制：CLNS【名字重复】IS-IS

OSPF为IP服务，IS-IS为CLNP网络服务。

这些都是不同组织在同时发展的，最后CONS阵营的方案得到公司的认可。

最后，ISO组织提出了OSI七层模型，但是CLNS的理念也得到认可，因此保留下来，叫做UDP

最后，IS-IS就感觉要消失了

这是为了统一行业标准。

OSPF虚链路：①克服孤岛区域：0，1，2区域不互相学路由

②两边区域0，中间区域1，但是两边区域0不互相学路由。

IS-IS没有这个问题。

外企推运营商用IS-IS 思科排第一

割接：干掉原有协议，换新协议

两种方法：①升级OSPF，使其拥有IS-IS特性

②直接用IS-IS取代OSPF

两种方法都不太科学……

所以我们现在使用的IS-IS，并不是纯IS-IS

集成IS-IS、双栈IS-IS

所以很多技术原来与IP对立，后来被IP吞并……

3：CLNP标识

3.1 NET 网络实体标识

类似于IP+MAC组合

3.2 NSAP 网络服务接入点

实现IP+MAC+端口号port_nam

NSAP在这个基础上把端口号加入。

3.3 CLNP标识格式

AFI A地址 F格式 I授权组织

数字由官方定好。47，48，49是用在局域网或私网的地址

所以官方有一张表。

0000 完整的一段，但有两块信息。前两个00代表域标识，后两个00代表区域标识

区域被域包含 域就是AS

巨型网络有不同的域标识

0001 0002 0003 同一域中的不同区域

sys-ID——MAC地址

所以说CLNP地址类似于IP+MAC，保证了全球唯一独立性

最后又有两个00 相当于端口号

CLNP全都是16进制 0A代表端口10

地址格式和MAC地址一样，但不是MAC地址

sys-id只要不一样就好，建议使用设备MAC地址……

看最后的部分是不是00

8.2）IS-IS中区域的特点

router isis

summary-address <汇总后网络> <汇总后网络掩码> level-1/level-2

15：ISIS 认证 及 ISIS 路由泄露

15.1）ISIS 认证

  1. 链路认证 isis password cisco

    该认证是利用 IIH 完成的，所以认证不同过影响邻接关系。链路认证是最高级别的，如果区域认证通过，链路认证不通过就是不通过。一般一个认证就行了。

  1. 区域认证 area-password cisco

    利用 LSP 完成的，认证不通过只影响路由的学习，不影响邻接关系。

  1. 域认证 domain-password cisco

    该认证如果不通过，只影响路由的学习，不影响邻接关系。

15.2）路由泄露

IGMP 互联网组管理协议

路由器及终端运行，帮助路由器发现直连子网中的多播接收者。

组（group）：所有对同一多播流感兴趣的成员组成了多播组。

组成员（group member）：组中的多播流接收者。

版本：IGMPv1、IGMPv2、IGMPv3

IGMPv1

  • 报告消息（report）：始发于终端，用于通告子网中的路由器存在特定组组成员。

    发送终端的IP 和MAC以及组信息。

  • 查询消息（request）：始发于路由器，用于查询直连子网中是否存在组成员。

    通常是路由器接收到多播流后触发。

一旦发现组成员，路由器就创造一个180s 的老化计时器，该计时器通过 report 消息进行维护。

IGMPv1的缺陷

  1. 响应时间慢

    组中没有成员时，路由器需要 180s 才能发现

  2. 存在多查询者

  3. 存在多转发者

IGMPv2

  • 报告消息（report）：和 v1 相同

  • 查询消息（request）：和 v1 相同

  • 离开消息（leave）：始发于终端，终端PC 不再是组成员时发送，用于通告路由器有组成员离开。

    收到离开消息时，立刻发送SQ，如下。

  • 特定查询消息（SQ）：始发于路由器，发送出去后设定的老化时间为 3s 。如果 3s 内收到 report ，说明发送离开消息的组成员并不是组中最后一个人；如果收不到任何消息，那么说明组成员不存在。

IGMPv2 查询者竞选机制：多个查询者通过比较查询消息源 ip 来确认谁最终负责查询，源 ip 越小越优。

多转发者缺陷 IGMPv2 无法克服。

多播路由器协议

是运行在路由器之间，负责建立多播路由路径，用于传递多播数据。

多播路由协议分类

  • 距离矢量：DVMRP 距离矢量多播路由协议（淘汰了）
  • 链路状态：MOSPF 多播的OSPF（非主流）
  • 距离向量：CBT 基于核心的树（非主流）
  • 独立于其他：PIM 协议独立多播（Cisco、主流）

PIM 的两个版本

PIMv1：用的是 IGMPv1，但是现在也用 IGMPv2

PIMv2：用的是 IGMPv2

PIM 的三种模式

  • 密集模式（dense mode）

    设备间带宽资源紧张，子网成员大部分是组成员的时候，建议使用密集模式。

    密集模式默认采用 IGMPv1。

  • 稀疏模式（sparse mode）

    设备间带宽资源依旧紧张，子网成员只有少数是组成员，建议使用稀疏模式。

    稀疏模式采用 IGMPv2。

  • 混合模式（sparse-dense mode）

    该模式下路由器先启用稀疏模式，稀疏模式失效后再用密集模式。

PIM 的密集模式详解

消息

  • 加入消息（join）：下游路由器通过加入消息将直连网络加入多播转发路径。
  • 修剪消息（prune）：下游路由器通过修剪消息，将原本隶属于多播路径的直连网络从多播路径中删除。
  • 接枝消息（graft）：下游路由器通过该消息请求将原本被拆除的多播路径再恢复过来。
  • 接枝确认消息（graft-ack）：上游路由器通过该消息告诉下游，直连路径恢复成多播转发路径。
  • assert消息（assert）：连接子网的路由器通过该消息，竞选出最终的转发者。

密集模式工作过程

扩散、剪枝。

PIM 的特殊机制

  • PIM 密集模式的 DR

    接口优先级+接口ip地址，构成竞选参数，越大越优。

    PIM密集模式早期使用IGMPv1，IGMPv1会出现多查询者问题，于是就利用DR的竞选，来帮助IGMPv1确认谁负责查询。

    PIM密集模式现在使用的是IGMPv2，DR就没用了。

  • PIM 消息中的 assert 作用

    通过竞选选出一个人负责转发数据。

    assert 消息，携带了三个竞选参数：

    • 当前路由器启用的底层协议管理距离，越小越优。
    • 当前路由器抵达多播源底层路由协议的开销，越小越优。
    • 当前路由器连接子网的接口 ip，越大越优。

PIM 密集模式的配置流程

  • 步骤1：确认参与多播的设备及接口

    所有需要接收多播流的接口和发送多播流的接口及连接组成员的接口，都要参与多播路由协议的启用。

  • 步骤2：路由器接口模拟成多播组成员

    注意：该命令可以反复录入，一个接口扮演多个组成员

  • 步骤3：路由器开启多播路由功能，接口启用多播密集模式

  • 步骤4：校验查看

    校验路由器是否发现了组成员：show ip igmp group

    校验路由器是否形成了PIM 邻居关系：show ip pim neighbor

    校验路由器多播路由表：show ip mroute

CEF

FIB 转发信息库（本身就是路由表的副本）

RIB 路由表（给FIB提供要素）

MPLS详解

multiple protocol label switch（多协议标签交换）

MPLS作用

MPLS是一种数据交换机制，基于标签库和数据携带的标签。

进行数据转发，同时支持多种协议。

MPLS在实际工作中，能实现如下两个基本功能：

  1. 数据包的高速转发
  2. 减少参与BGP的设备数量、减少对等体关系、减少不必要的路由学习

MPLS术语

label 标签

总长 32bit 由如下几个部分组成：

前20bit 是标签取值范围

第21bit 是栈底位，栈底位为1则表明该标签已经是最后一个

第22~24bit 是体验位，可以将标签对应的数据分成8个级别，给予不同的传输策略。

最后8bit 是TTL，限制携带标签的数据传递范围。

MPLS 模式

帧模式：frame mode（主流模式），标签被插在帧头的后面，包头前面。

信令模式：cell mode 标签被插在信令头后面。

LSR：标签交换路由器

启用了MPLS的路由器统称LSR。

入站 LSR （inbound LSR）：接收到 IP 数据包，压入标签，传递给 MPLS 网络。

链路中 LSR （link LSR）：接收到携带标签的数据，基于标签进行数据转发。

出站 LSR （outbound LSR）：负责将标签移除，并且变回 IP 数据报文。

压入、交换、移除、弹出

压入：入站LSR将标签压入数据。

交换：链路中LSR基于标签库，进行标签的置换、数据的传递。

移除：出站LSR将数据携带的所有标签全部删除。

弹出：出站LSR将数据携带的栈顶标签弹出，其他标签保留。

LSP：链路标签路径

是MPLS设备传递数据所需的路径。（即路由）

LSP是通过标签的关联产生的。

标签分配、分发、关联、维护协议

TDP：标记分发协议、思科私有

LDP：标签分发协议、公有公开

LDP详解

LDP作用

  1. 利用 hello 消息建立 LDP 邻接关系。
  2. 基于CEF给本地网络分配本地标签。
  3. 将网络对应的标签分发共享给邻居
  4. 基于底层协议完成本地标签与远程共享标签的关联，最终形成LSP。
  5. 负责监控LSP的有效性，并且维护它。

LDP邻接关系的建立

LDP 邻居的发现，通过hello消息，基于UDP，目标端口646。

LDP 利用TCP，目标端口646完成邻接关系建立。

LDP可以设定RID，RID扮演更新源，更新源会成为对端TCP的目标接收地址，所以更新源必须可达。

LDP标签的分配

除BGP路由，LDP可以给本地直连网络以及非直连网络分配标签。

LDP标签的共享

两种形式：下游主动、上游主动。

实际上标签的分享是给所有的邻居，不遵循所谓的上下游。

LDP标签的关联

LDP永远将路由路径下一跳共享的标签与对应网络的本地标签关联。

boundaddress：绑定地址，参与MPLS设备的IP会成为绑定地址和，会跟着本地标签一起共享给LDP邻居，邻居根据底层协议下一跳地址和绑定地址的判断来确认哪个LDP邻居是下一跳，再绑定。

LDP的特殊标签

  • UNTAGE：未标记

出现原因：底层是ospf,本地网络无法找到对应的远程标签，LDP给网络分配untage。

处理方式：目标网络远程标签如果是untage，那么LSR会将所有标签移除，向外传递。

  • POP：倒数第二跳标签

出现原因：当前路由器给本地直连网络分配的本地标签就是POP。

处理方式：如果出站标签是POP，那么LSR 会将栈顶标签弹出。

意义：提高数据转发效率，减少一次标签库查找。

LDP的两张表

LIB：标签信息库

目标网络本地标签和学习到的远程标签存储在这里。（未关联）

LFIB：标签转发信息库

标签关联后的结果 即 LSP 存储在这里。

CCIE bgp

边界网关路由协议

BGP简介

工作类型：路径矢量路由选择协议

基于tcp(三次握手)

三张表：邻居表（peer表） bgp转发表 路由表

BGP版本

BGPv4：IPv4

BGPv4+：IPv6

MP-BGP：协议npls

M-BGP：多播

BGP优缺点

优点

1EGP路由表只能存储千条信息，bgp可上万条。

2丰富的路径属性，13条选路原则，提供了更丰富的路由操控策略。

缺点

1对路由器性能要求较高。

2配置程序比较复杂。

3一般小网络不建议使用bgp

AS 和 邻居类型

bgp AS 一台路由器只能属于一个

公网AS1-65411 65412-65535私有AS

iBGP：处于同一AS内的BGP路由器所建立的BGP对等体关系

eBGP：不同AS内的BGP路由器建立的…

###

BGP消息

open：建立成功，会互相发送open信息(类似hello)协商邻居关系的各项参数，等待对方确认。

keeplive：维持对等体(类似邻居)关系的建立，60s发一次，180s后peer关系结束

update：不会周期性更新，只会增量更新或触发更新。

Notification：当BGP检测到错误状态时，就向对等体发出Notification消息，之后BGP连接会立即中断。

peer关系建立过程

IDLE（空闲状态）：本地路由向外发送tcp(端口179)请求，等回复

connect状态：tcp建立

active：建立不成功，再次请求tcp连接的建立（回到IDLE）。

open sent：建立成功，发送open消息

openconfirm：收到对方的信息进行参数对比，参数一致则建立完成。

established：连接建立。。update。。keepalive。。

bgp的防环机制

iBGP-水平分隔原则

BGP对等体路由器收到iBGP路由，不会发送给其他iBGP对等体路由器。

但是纯水平拓扑上该原则会导致路由不被传递学习。两个解决方案

全互联拓扑解决方案：所有路由器间都建立iBGP关系。（垃圾方案）

路由反射器解决方案：

eBGP

基于AS PATH信息的检测。接收到来自于eBGP peer的路由更新，更新所带AS-PATH 信息包含了当前设备AS号，则该路由更新忽略。

特殊拓扑下路由反射器的防环机制

originate id：起源id。路由器将设备的RID设置为起源ID。起源ID和当前设备RID一样，则路由更新不学习。

cluster id：簇id。管理员在路由反射器上设置，路由更新携带的簇ID一致，则路由更新不学习。

同步与下一跳判断

路由更新来自于iBGP peer时才考虑同步和下一跳。

同步机制

对于当前启用BGP的设备，路由更新对应的网络底层协议也必须能学习到，这才叫同步。同步是为了避免路由黑洞。

现实中同步被关闭，路由黑洞问题无法避免，但利用MPLS能解决。

bgp路由黑洞

路由更新的传递设备也就是数据传输的沿途设备，并没有参与BGP ，学习该路由，所以数据抵达后，只能丢包。

解决方法：

1重分发(不建议，会增大路由表)

2全互联的网络拓扑(两两之间建立ibgp对等体)

下一跳判断

来自于iBGP peer的路由更新，如果下一跳不可达，路由更新不学习。

路由源于AS外，通告者是eBGP peer，下一跳就是eBGP peer地址。

路由源于AS外，通告者是iBGP peer，下一跳依旧是eBGP peer地址。此时下一跳可能不可达。

路由源于iBGP peer，通告者是iBGP peer，下一跳就是iBGP peer地址。

BGP的路由更新属性

  1. 公认必选

    NEXT-HOP：下一跳

    AS-PATH：AS路径信息

    Originate code：起源代码

    ​ 描述了目标网络进入BGP的方式（I E ?）

    ​ I ：目标网络源于底层协议通过network进入BGP

    ​ ?：外部路由以重分发的方式进入BGP

  2. 公认任选（自选）

    LP：本地优先

    ​ 路由更新在IBGP peer间传递时，LP值才出现，且为100

    ​ 作用：影响iBGP peer 出站路径的选择，LP参数越大越优。

    automatic aggregate：原子聚合

    ​ BGP发生了自动聚合或手动聚合时，聚合后网络携带该属性。

    ​ 原子聚合属性用于提醒更新接受者，路由更新目标网络可能丢失了部分原有属性。

  3. 任选可传递

    Aggregated：被聚合属性——聚合后的网络携带的属性，表明该网络是聚合来的。

    Community：团体属性 是携带特定策略的标记

    ​ 公认团体属性：

    ​ Internet：所有的BGP路由默认携带该属性，该属性对应的策略是，共享给所有的BGP peer。

    ​ no-export：不导出，携带该属性的路由更新不共享给eBGP peer。

    ​ no-advertise：不通告，携带该属性的路由更新不共享给任何BGP peer。

    ​ local-as：本地as，携带该属性的路由更新不共享给联盟内其他eBGP peer。

    ​ 自定义团体属性：

    ​ AA:NN

    ​ AS号：标记值

  4. 任选不可传递（属性保留，但值为空）

    MED：多出口距离

    ​ eBGP间路由共享携带该属性，MED值用于影响eBGP peer入站路径选择。MED值默认为0，越小越优。

BGP选路原则

选路原则决定了BGP按什么标准进行路由更新属性的比较

  1. 优选拥有最高权重值的路径 weight

    用于影响当前路由器出站路径选择

    默认32768，越大越优

    不同类型的对等体出现，需要影响路径抉择，权重更优。

  2. 优选LP大的

  3. 优选起源于本地的路径

    next-hop=0.0.0.0

  4. 优选拥有最短AS-PATH 路径信息的

  5. 比较Origin属性<IGP优先于EGP优先于Incomplete>

  6. 优选拥有最佳MED值的

  7. eBGP peer 的路径优于 iBGP peer的路径

  8. 优选离下一跳更近的路径

  9. 针对eBGP peer的路由，通过命令行允许多条路径，成为最佳进入路由表

  10. 优选最早被学习的路由，只针对 eBGP peer

  11. 优选拥有最小RID的BGP peer提供的路由

  12. 优选拥有最短cluster-id 的路径

  13. 优选拥有最低peer地址的邻居提供的路径

自动汇总 auto-summary

自动汇总用来设定BGP宣告是精确还是不精确的

auto-summary 关闭的情况下，BGP 处于精确宣告状态

auto-summary 开启的情况下，BGP 处于非精确宣告状态

被宣告的网络只要拥有本地对应的子网，被宣告网络就可以当做更新发送出去。

针对外部路由，auto-summary 开启，外部路由自动汇总，否则外部路由就明细进入。
