CCIE 交换机基础

交换机基础

泛洪：除接收端口外，所有端口都向外去发。

mac地址表=cam表

广播域：所能传播的最大范围

冲突域：发生冲突之后所影响的范围

CSMA/CD：载波帧听多链路访问

交换机指示灯：

绿色常亮：接口两端没有问题，但是当前没有数据从中通过

绿色不停地闪：正在传输数据

橘红色慢闪：端口没问题,对端线路或者对端的端口有问题

橘红色快闪：端口和对端连接没有问题，正在协商参数建立

黄色闪动：物理接口有问题，尝试重启。

不闪： 物理接口有问题，无法通信

vlan（虚拟局域网）

作用：分隔广播域

所有接口默认属于vlan1（默认vlan，不可删除）。

vlan ID范围和解释

\1. 0和4095 仅仅限于系统使用 用户不能查看和使用

\2. 1 正常 cisco默认的vlan,用户可以使用但是不能删除

\3. 2–1001 正常 用于以太网的VLAN，用户可以创建。使用。删除。

\4. 1002–1005 正常 用于FDDI和令牌环的CISCO默认VLAN，用户不能删除

\5. 1006–1024 保留 仅仅限于系统使用。用户不能查看和使用。

\6. 1025–4094 扩展 仅仅用于以太网VLAN

vtp（虚拟传输协议）

用于同步同一网络中交换机上网络的信息（减小庞大的配置量）

BGP 的路由更新过滤

route-map、prefix-list、filter-list（筛选列表）

利用 route-map

可以是 in 方向，也可以是 out 方向。

route-map 有隐藏的 deny all。

利用 prefix-list

route-map 不光可以完成过滤，还可以对放行的网络设置策略。

prefix-list 只能完成过滤。

prefix-list 也有默认的 deny all。

prefix-list 的放行所有 ip prefix-list x permit 0.0.0.0/0

利用 正则表达式

正则表达式匹配击中的是 AS 路径信息。

放行所有：ip as-path <id> permit _

BGP 的公认团体属性限制路由更新的发送

路由更新携带 no-advertise，路由更新被接收后不发送给任何 BGP peer。

路由更新携带 no-export 或 local-as，路由更新被接收后不发送给任何 eBGP peer。

CCIE 防火墙

防火墙

防火墙的分类

硬件防火墙：流量的省查及访问的控制：保护的是网络

软件防火墙：省查木马或病毒：保护的是系统

应用防火墙：WAF（WEB 应用防火墙）、APP 防火墙

硬件防火墙

checkpoint（诺基亚）

netscreen（詹博、Juniper）

PIX、ASA（Cisco）SDN（安全自防御网络、软件定义的网络）

华为、天融信

硬件防火墙的部署

网络的边缘部署防火墙，一般放在 PPPOE 拨号设备的身后，关键节点之间。

防火墙的基础知识

防火墙工作模式

路由模式：（默认模式）防火墙读取 IP 包头，进行网络层的保护，支持动态路由协议和静态路由。

交换模式：

防火墙的多模式和单模式

多模式支持虚拟防火墙，即把一台防火墙配置成多台逻辑防火墙。

单模式是默认模式。

防火墙接口的类型

inside：内部接口

通常连接的是信任网络即内网

outside：外部接口

通常连接的是非信任公共网络，比如 ISP 或 其他组织网络

DMZ：非军事区域

通常数据中心放在 DMZ 区域

防火墙接口等级和访问规则

inside 接口安全等级：100

outside 接口安全等级：0

DMZ 接口安全等级：50、0

高等级接口接收流量穿越防火墙从低等级接口发送出去，必须经过 NAT。

低等级接口接收流量穿越防火墙从高等级接口发送出去，默认不允许。

除非是返回流量或被管理员手动放行。

相同安全级别的接口相互通讯，默认不允许通讯，除非管理员放行。

配置

  1. 查看接口show interface ip brief
  1. 查看特定配置`show run

  1. 查看转换表： show xlate

  2. 配置接口

  3. 注入静态路由

  4. 防火墙上配置 ACL

  5. 防火墙上配置 NAT

    1. 创建对象网络，添加允许转化的内网信息

      防火墙可以创建多个对象网络，一个对象网络对应到一个部门。

    2. 创建对象组，将对象网络关联并且设置 NAT 转化语句。

      ！！补充知识

      NAT：网络地址转换

      静态 NAT：管理员指定的一对一转化

      动态 NAT：路由器自动的完成一对一转化

      端口 NAT：PAT 多对一的转化，利用端口号区分开

      ！！注意

      防火墙默认不允许 PING 包流量从低等级接口穿越防火墙抵达高等级接口。

    3. 内网服务发布到外网

      1. 定义对象网络，把服务器地址放入

      2. 完成静态 NAT 的配置发布服务

      3. 利用 ACL 放行来自外网主动访问内网服务的流量

CCIE IPSEC VPN

IPSEC VPN

Ip security vpn，是一个工作在网络层的 VPN 技术。

数据在公共网络上面临的风险

  1. 不加密，明文传递
  2. 没有完整性保护，数据容易被篡改
  3. 没有对等体认证，容易受到中间人攻击
  4. 抵赖

IPSEC VPN 解决的问题

  1. 对等体认证，数据发送之前必须经过认证且通过

    口令认证、数字证书（计算机世界的身份证）、加密随机数

  2. 数据加密

    des、3des、aes

  3. 数据完整性保护

    HMAC：哈希消息认证码

  4. 数字签名，防抵赖

    DSA（RSA 公司开发）

KEY：密匙

KEY 是一个工具，可以参与算法作为变量出现，也可以参与认证作为口令出现。

对称密钥、非对称密钥

（加密算法中的 KEY 的理解）

  1. 对等密钥及对称密钥加密算法

    加密解密密钥是同一把

    坏处：不够安全、Key 的共享存在风险

    好处：简单高效

  2. 非对称密钥及非对称密钥算法

    非对称密钥是一对KEY，公钥和密钥。

    公钥最后会公开，密钥不公开。

    公钥加密的数据，只有私钥可以解开。

    私钥加密的数据，只有公钥可以解开。

    坏处：复杂，数据会变庞大

    好处：安全可靠

IPSEC VPN 的加密密钥

非对称密钥共享是用对称密钥及算法保护

数据本身是用非对称密钥及算法保护

密钥产生过程和共享方法

DH 算法负责产生密钥和共享密钥

IPSEC VPN 的关键组件

ISAKMP：互联网安全关联密钥管理协议

  • ISA：通讯双方建立了安全连接就叫做一个 SA 成功了。

    IPSEC VPN 的 SA 有两个：一个是保护 VPN 的，一个是保护数据的。

  • KMP：KMP 是通过 IKE（互联网密钥交换机制）完成的，IKE 专门负责产生密钥，共享密钥。

    IKE 分两个阶段：阶段1产生和共享VPN自身所需的密钥，阶段2利用阶段1的密钥算出新密钥，用于保护用户数据。

ESP 和 AH

ESP 和 AH 都是 VPN 头部信息的缩写

帧头、包头、段头、数据、帧尾

数据经历 VPN 保护后变化成如下：

帧头、VPN 头、包头、段头、数据、帧尾

VPN 头就分两种：

  1. ESP：封装安全静荷

    该头负责保护数据的完整性，同时加密数据

  2. AH：认证头

    该头只负责保护数据完整性，不加密数据

IPSEC VPN 的模式

  1. 隧道模式（tunnel）

    VPN 网关通过一个隧道模式的 VPN 连接，保护身后所有的通讯数据。

  2. 传输模式（transport）

    通讯双方自身建立 VPN 连接，保护通讯数据。

区别：

隧道模式的新包头，目的 IP 和源 IP 是 VPN 的对等体地址。

传输模式的新包头，目的 IP 和源 IP 复制了原有的 IP 地址。

NAT 对 IPSEC VPN 的影响

VPN 流量几乎永远是先被 inside 接口接收，所以 NAT 会直接修改 IP 包头，导致 VPN 无法识别感兴趣流。

IS-IS（中间系统到中间系统）

数据传递方式

  • 面向连接 tcp/ip

  • 非面向连接 clns/clnp 非连接的网络服务(负责控制数据传递过程)/非连接的网络协议(网络地址标识)

    最后，ip获得广泛使用，但是CLNS的理念也得到认可，因此保留下来，叫做UDP。

IS-IS术语

  • IS：中间系统 即 路由器router

  • ES：终端系统 （但是现在已经没有使用clnp标识的设备了）

  • clnp：标识

    NET：网络实体标识（网络+主机）128bit

    clnp：clnp地址 NSAP网络服务应用标识

clnp地址解释

49.0001.0000.0000.0001.00

49.0002.0000.0000.0002.00

49.0101.0000.0000.0001.00

49：AFI地址格式标识 A地址 F格式 I授权标识 数字由官方的表定好。47，48，49用于私网。

0101 01/01 域/区域 （区域包含在域内）这里为两个十六进制数

0000.0000.0001 mac地址

00 服务号（作用类似于端口号）

IS-IS 域、区域{骨干区域，常规区域}

域：AS

区域：0，1，2 ……

路由器角色类型

  • OSPF{角色：DR BDR DR-other，类型：ABR ASBR}
  • IS-IS {角色：Dis(不完整)，类型：L1 L2 L1/L2}

伪节点

  • OSPF：DR{减少邻接关系，提高更新效率}
  • IS-IS：Dis

IS-IS类型

  • L2 骨干区域
  • L1 常规区域
  • L1/L2(默认类型) 连接两个区域

OSPF LSA 特殊区域

LSA：Link-State Advertisement 链路状态广播

DBD：数据库描述报文

LSR：链路状态查询

LSU：链路状态更新

OSPF七种LSA类型

  • LSA1（Router LSA）

    起源：区域内每台设备都能产生LSA1

    内容：节点信息{RID network cost}

    扩散：不允许穿越ABR，也就是区域内扩散。

    路由：区域内路由

  • LSA2（Network LSA）

    起源：只有DR可以产生

    内容：拓扑信息{节点的直连情况，谁是DR}

    扩散：区域内

  • LSA3（Network summary LSA）

    起源：ABR

    内容：区域网络信息

    扩散：整个OSPF AS

    路由：区域间路由

  • LSA4（ASBR summary LSA）

    起源：ASBR所在域的ABR

    内容：ASBR的RID

    扩散：整个OSPF

    路由：外部{OE1，OE2}

  • LSA5（Autonomous system external LSA）

    起源：ASBR

    内容：外部路由

    扩散：整个OSPF

    路由：外部{OE1，OE2}

ISIS的汇总及重分发

汇总和重分发最好做在拥有L2角色的设备上。

L1设备上完成重分发和汇总的解决方法

    重分发时指定外部路由类型改为L1

  • L1设备上做汇总

    被汇总的网络必须以外部路由形式进入ISIS，再汇总才生效

    ！！注意

    ISIS的外部路由类型是L2，那么接受者不累加接口开销

ISIS的路由泄露

ISIS路由泄露 指 将指定的L2数据库目标网络信息转化成L1的，发送给L1设备。

路由泄露的配置流程

步骤3：L1/L2 设备上做 ISIS 到 ISIS 的重分发，并且指定是利用 route-map 完成 L2 数据库条目到 L1 数据库条目的泄露。

gcc常用标签

对于C/C++编写的程序，从源代码到可执行文件，一般经过下面四个步骤：

  1. 预处理，产生.ii文件

    gcc -E hello.c > hello.i 只进行预处理，会默认输出到标准输出。

  2. 编译，产生汇编文件(.s文件)

    gcc -O1 -S hello.c -S 产生汇编代码(注意是大写！！) hello.s ；-O1是程序优化等级，O2优化比O1高，查看汇编代码时使用低优化等级有助于阅读学习，高版本gcc可以使用-Og

  3. 汇编，产生目标文件(.o或.obj文件)

    gcc -O1 -c hello.c 汇编，阻止编译器创建一个完整的程序，产生二进制目标代码文件hello.o。

  4. 链接,产生可执行文件(.out或.exe文件)

    gcc -o hello hello.c

JSON

JavaScript 对象表示法

JSON 是存储和交换文本信息的语法。类似 XML。

JSON 比 XML 更小、更快，更易解析。

JSON 使用 JavaScript 语法来描述数据对象，但是 JSON 仍然独立于语言和平台。JSON 解析器和 JSON 库支持许多不同的编程语言。

JSON - 转换为 JavaScript 对象

JSON 文本格式在语法上与创建 JavaScript 对象的代码相同。

由于这种相似性，无需解析器，JavaScript 程序能够使用内建的 eval() 函数，用 JSON 数据来生成原生的 JavaScript 对象。

为什么使用 JSON？

对于 AJAX 应用程序来说，JSON 比 XML 更快更易使用：

  1. 使用 XML
  • 读取 XML 文档
  • 使用 XML DOM 来循环遍历文档
  • 读取值并存储在变量中
  1. 使用 JSON
  • 读取 JSON 字符串
  • 用 eval() 处理 JSON 字符串

JSON 语法规则

JSON 语法是 JavaScript 对象表示法语法的子集。

  • 数据在名称/值对中
  • 数据由逗号分隔
  • 花括号保存对象
  • 方括号保存数组

JSON 值可以是：

  • 数字（整数或浮点数）
  • 字符串（在双引号中）
  • 逻辑值（true 或 false）
  • 数组（在方括号中）
  • 对象（在花括号中）
  • null

JSON 解析器

提示：eval() 函数可编译并执行任何 JavaScript 代码。这隐藏了一个潜在的安全问题。

使用 JSON 解析器将 JSON 转换为 JavaScript 对象是更安全的做法。JSON 解析器只能识别 JSON 文本，而不会编译脚本。

在浏览器中，这提供了原生的 JSON 支持，而且 JSON 解析器的速度更快。

较新的浏览器和最新的 ECMAScript (JavaScript) 标准中均包含了原生的对 JSON 的支持。
