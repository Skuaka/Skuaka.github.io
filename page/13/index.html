CCIE DM-VPN

DM-VPN

动态多点VPN、Cisco私有

IPSEC VPN 存在的问题

  1. 不支持多播
  2. 复杂的拓扑

DM-VPN 的术语

  • M-GRE（多点隧道）一点对多点
  • 物理 IP 地址（物理接口）/逻辑 IP 地址（tunnel 私有 IP）
  • NHRP（下一跳解析协议、总部和分部都启用）
    NHS（下一跳服务器、总部）

DM-VPN 的三个阶段

阶段一

  • M-GRE：EIGRP
  • 分支和分支要通过总部才能相互通讯

阶段二

  • M-GRE：EIGRP
  • 分支和分支可以直接通讯

阶段三

  • 单云双中心（两个 HUB 同时工作互为备份，还是依赖 DM-VPN 阶段2）

DM-VPN 的基础配置流程

步骤1：完成 HUB 上 M-GRE 的配置

步骤2：完成 spoke 上的 M-GRE 配置

步骤3：完成 HUB 上隧道 NHRP 的配置

步骤4：完成 spoke 上 NHRP 的配置

步骤5：隧道逻辑 IP 地址参与 EIGRP

步骤6：完成 IPSEC 的配置并且套用到隧道上

杂：

查看 HUB 的分支信息：show ip nhrp

关闭 EIGRP 水平分割：no ip split-horizon eigrp 1

开启 NHRP 捷径：ip nhrp shortcat

单云双中心的配置细节

区别1：两个 HUB 需要互做 NHRP 的映射。

区别2：两个 spoke 需要设定两个 NHS 服务器，并且谁先被设定谁就是主设备。

CCIE 跨域的多播

跨域的多播

源和组成员在不同的AS，需要传递多播数据流。

跨域多播的问题和挑战

组成员无法发现源、源无法发现组成员、多播树无法建立。

MSDP 多播源发现协议

MSDP 是配置在不同 AS RP 上的，它会建立基于 TCP 的对等体关系，利用自身的 SA（源关联）消息共享”源地址信息、多播组地址信息、RP 信息”给对端 RP。

MSDP 配置流程

步骤1：RP上配置新的 loopback 接口，并且宣告进 BGP。

步骤2：RP上完成 MSDP 的配置

排错手段

ip 路由疑难解析

多播排错

ping 多播组地址 不同

  1. 确认发现组成员

    show ip igmp group

    查看组地址、本地接口、报告者 是否出现

    错点

    • 没有模拟成组成员，缺乏接口下：ip igmp join-group <a.b.c.d>
    • 路由器接口没有加入PIM，缺乏接口下:ip pim <x>
    • 连通性问题

  2. 确认稀疏模式获取 RP 信息

    show ip pim rp mapping 查看 RP 映射，查看 RP 的结果

    show ip pim rp 查看多播组和 RP 的关联

    错点

    • 没有通过静态 RP 指明 RP 地址
    • BSR 的 C-BSR 或 C-RP 配置问题
    • auto-rp 的配置问题
    • 缺乏auto-rp 侦听的配置
    • 设备间缺乏必要的PIM 邻居：show ip pim neighbor
      • 设备缺乏多播路由功能和接口忘记加入PIM
      • 接口忘记激活、错误的安全策略挡住了PIM消息
    • 设备无法正常抵达 RP 地址，所以不显示获取的 RP 消息
      • 路由层面问题、忘记宣告、忘记重分发、错误的过滤

  3. RPF 检测未通过

    • 缺乏必要的更新源路由
    • 忘记宣告、忘记重分发、错误的路由更新过滤

  4. 是否是跨域多播、缺乏必要的配置和连通性

    • 缺乏必要的 MSDP 配置和 MSDP 对等体关系

      show run | sec ip msdp

      show ip msdp summary

      MSDP peer 地址不可达

      MSDP 链接源和起源 ID 设置错误

    • RPF 检测不通过

  5. 错误的安全策略

结束

CCIE 防火墙

防火墙

防火墙的分类

硬件防火墙：流量的省查及访问的控制：保护的是网络

软件防火墙：省查木马或病毒：保护的是系统

应用防火墙：WAF（WEB 应用防火墙）、APP 防火墙

硬件防火墙

checkpoint（诺基亚）

netscreen（詹博、Juniper）

PIX、ASA（Cisco）SDN（安全自防御网络、软件定义的网络）

华为、天融信

硬件防火墙的部署

网络的边缘部署防火墙，一般放在 PPPOE 拨号设备的身后，关键节点之间。

防火墙的基础知识

防火墙工作模式

路由模式：（默认模式）防火墙读取 IP 包头，进行网络层的保护，支持动态路由协议和静态路由。

交换模式：

防火墙的多模式和单模式

多模式支持虚拟防火墙，即把一台防火墙配置成多台逻辑防火墙。

单模式是默认模式。

防火墙接口的类型

inside：内部接口

通常连接的是信任网络即内网

outside：外部接口

通常连接的是非信任公共网络，比如 ISP 或 其他组织网络

DMZ：非军事区域

通常数据中心放在 DMZ 区域

防火墙接口等级和访问规则

inside 接口安全等级：100

outside 接口安全等级：0

DMZ 接口安全等级：50、0

高等级接口接收流量穿越防火墙从低等级接口发送出去，必须经过 NAT。

低等级接口接收流量穿越防火墙从高等级接口发送出去，默认不允许。

除非是返回流量或被管理员手动放行。

相同安全级别的接口相互通讯，默认不允许通讯，除非管理员放行。

  3. 注入静态路由

  4. 防火墙上配置 ACL

  5. 防火墙上配置 NAT

    1. 创建对象网络，添加允许转化的内网信息

      防火墙可以创建多个对象网络，一个对象网络对应到一个部门。

    2. 创建对象组，将对象网络关联并且设置 NAT 转化语句。

      静态 NAT：管理员指定的一对一转化

      动态 NAT：路由器自动的完成一对一转化

      端口 NAT：PAT 多对一的转化，利用端口号区分开

      ！！注意

      防火墙默认不允许 PING 包流量从低等级接口穿越防火墙抵达高等级接口。

    3. 内网服务发布到外网

      1. 定义对象网络，把服务器地址放入

      2. 完成静态 NAT 的配置发布服务

      3. 利用 ACL 放行来自外网主动访问内网服务的流量

CCIE BGP 补充知识

BGP 的路由更新过滤

route-map、prefix-list、filter-list（筛选列表）

利用 route-map

可以是 in 方向，也可以是 out 方向。

route-map 有隐藏的 deny all。

利用 prefix-list

route-map 不光可以完成过滤，还可以对放行的网络设置策略。

prefix-list 只能完成过滤。

prefix-list 也有默认的 deny all。

prefix-list 的放行所有 ip prefix-list x permit 0.0.0.0/0

利用 正则表达式

正则表达式匹配击中的是 AS 路径信息。

放行所有：ip as-path <id> permit _

BGP 的公认团体属性限制路由更新的发送

路由更新携带 no-advertise，路由更新被接收后不发送给任何 BGP peer。

路由更新携带 no-export 或 local-as，路由更新被接收后不发送给任何 eBGP peer。

查看BGP转发表中携带特定团体属性的路由更新show ip bgp community <no-export/no-advertise>

BGP 自定义团体属性标记及策略设置

自定义团体属性格式：

AA：NN（AA 是 AS 号，NN 是标记值 0~65440）

自定义团体属性的配置步骤

步骤1：更新源处打上标记

IPSEC VPN

Ip security vpn，是一个工作在网络层的 VPN 技术。

数据在公共网络上面临的风险

  1. 不加密，明文传递
  2. 没有完整性保护，数据容易被篡改
  3. 没有对等体认证，容易受到中间人攻击
  4. 抵赖

IPSEC VPN 解决的问题

  1. 对等体认证，数据发送之前必须经过认证且通过

    口令认证、数字证书（计算机世界的身份证）、加密随机数

  2. 数据加密

    des、3des、aes

  3. 数据完整性保护

    HMAC：哈希消息认证码

  4. 数字签名，防抵赖

    DSA（RSA 公司开发）

KEY：密匙

KEY 是一个工具，可以参与算法作为变量出现，也可以参与认证作为口令出现。

对称密钥、非对称密钥

（加密算法中的 KEY 的理解）

  1. 对等密钥及对称密钥加密算法

    加密解密密钥是同一把

    坏处：不够安全、Key 的共享存在风险

    好处：简单高效

  2. 非对称密钥及非对称密钥算法

    非对称密钥是一对KEY，公钥和密钥。

    公钥最后会公开，密钥不公开。

    公钥加密的数据，只有私钥可以解开。

    私钥加密的数据，只有公钥可以解开。

    坏处：复杂，数据会变庞大

    好处：安全可靠

IPSEC VPN 的加密密钥

非对称密钥共享是用对称密钥及算法保护

数据本身是用非对称密钥及算法保护

密钥产生过程和共享方法

DH 算法负责产生密钥和共享密钥

IPSEC VPN 的关键组件

ISAKMP：互联网安全关联密钥管理协议

  • ISA：通讯双方建立了安全连接就叫做一个 SA 成功了。

    IPSEC VPN 的 SA 有两个：一个是保护 VPN 的，一个是保护数据的。

  • KMP：KMP 是通过 IKE（互联网密钥交换机制）完成的，IKE 专门负责产生密钥，共享密钥。

    IKE 分两个阶段：阶段1产生和共享VPN自身所需的密钥，阶段2利用阶段1的密钥算出新密钥，用于保护用户数据。

ESP 和 AH

ESP 和 AH 都是 VPN 头部信息的缩写

帧头、包头、段头、数据、帧尾

数据经历 VPN 保护后变化成如下：

帧头、VPN 头、包头、段头、数据、帧尾

VPN 头就分两种：

  1. ESP：封装安全静荷

    该头负责保护数据的完整性，同时加密数据

  2. AH：认证头

    该头只负责保护数据完整性，不加密数据

IPSEC VPN 的模式

  1. 隧道模式（tunnel）

    VPN 网关通过一个隧道模式的 VPN 连接，保护身后所有的通讯数据。

  2. 传输模式（transport）

    通讯双方自身建立 VPN 连接，保护通讯数据。

区别：

隧道模式的新包头，目的 IP 和源 IP 是 VPN 的对等体地址。

传输模式的新包头，目的 IP 和源 IP 复制了原有的 IP 地址。

同时进行的，没有谁早谁晚的说法。

15：ISIS 认证 及 ISIS 路由泄露

15.1）ISIS 认证

  1. 链路认证 isis password cisco

    该认证是利用 IIH 完成的，所以认证不同过影响邻接关系。链路认证是最高级别的，如果区域认证通过，链路认证不通过就是不通过。一般一个认证就行了。

  1. 区域认证 area-password cisco

    利用 LSP 完成的，认证不通过只影响路由的学习，不影响邻接关系。

  1. 域认证 domain-password cisco

    该认证如果不通过，只影响路由的学习，不影响邻接关系。
